Django - Corso Completo

Django è un framework web Python ad alto livello che incoraggia lo sviluppo rapido e un design pulito e pragmatico.

Perché Django?

  • **Veloce**: Progettato per portare le applicazioni dalla bozza al completamento il più rapidamente possibile
  • **Sicuro**: Aiuta gli sviluppatori ad evitare molti comuni errori di sicurezza
  • **Scalabile**: Usato da alcuni dei siti più trafficati al mondo

    • Cosa puoi costruire con Django?

  • Siti web dinamici
  • REST API
  • Applicazioni web complesse
  • CMS (Content Management System)

    • Esempio: Una view Django

    
from django.http import HttpResponse

def home(request):
    return HttpResponse("Ciao, mondo!")

    Introduzione

    Introduzione a Django

    Django è stato creato nel 2003 da Adrian Holovaty e Simon Willison mentre lavoravano in un giornale online.

    Caratteristiche principali

    MTV Pattern

    Django segue il pattern **MTV** (Model-Template-View):

  • **Model**: Gestisce i dati e la logica del database
  • **Template**: Gestisce la presentazione (HTML)
  • **View**: Gestisce la logica tra Model e Template

    • "Batterie incluse"

    Django viene con molte funzionalità pronte all'uso:

  • Pannello di amministrazione automatico
  • ORM (Object-Relational Mapping)
  • Sistema di autenticazione
  • Sistema di form
  • Internazionalizzazione

    • Versioni Django

    
Django 4.2 LTS (Long Term Support)
Django 5.0+ (versione corrente)

    Prerequisiti

  • Python 3.8+
  • Conoscenza base di Python
  • Conoscenza base di HTML/CSS
  • Concetti di database

    Models

    Models in Django

    I **models** definiscono la struttura del database.

    Definizione model

    
# members/models.py
from django.db import models

class Member(models.Model):
    nome = models.CharField(max_length=100)
    cognome = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    data_iscrizione = models.DateField(auto_now_add=True)
    attivo = models.BooleanField(default=True)

    def __str__(self):
        return f"{self.nome} {self.cognome}"

    class Meta:
        ordering = ['cognome', 'nome']

    Introduzione

    Introduzione ai QuerySet Django

    I QuerySet permettono di leggere, filtrare e manipolare dati dal database.

    Cos'è un QuerySet?

    Un QuerySet è una **raccolta di oggetti** del database. È "lazy": le query SQL vengono eseguite solo quando i dati vengono effettivamente richiesti.

    Recupera tutti gli oggetti

    
from members.models import Member

# Recupera tutti i membri (QuerySet)
tutti = Member.objects.all()

# Iterazione (esegue la query SQL)
for m in tutti:
    print(m.nome)

# Conta senza recuperare i dati
numero = Member.objects.count()

    Metodi principali

    
# Tutti i record
Member.objects.all()

# Un singolo record (solleva eccezione se non trovato)
Member.objects.get(pk=1)
Member.objects.get(email='mario@example.com')

# Filtro (restituisce QuerySet)
Member.objects.filter(attivo=True)

# Escludi
Member.objects.exclude(attivo=True)

# Ordinamento
Member.objects.order_by('cognome')
Member.objects.order_by('-data_iscrizione')  # Decrescente

# Limita risultati
Member.objects.all()[:10]    # Primi 10
Member.objects.all()[5:10]   # Dal 6° al 10°

    SQL generato

    
qs = Member.objects.filter(attivo=True).order_by('cognome')
print(qs.query)
# SELECT * FROM members_member WHERE attivo=True ORDER BY cognome

    QuerySet - Metodo Filter

    `filter()` recupera tutti i record che corrispondono ai criteri specificati.

    Filtri base

    
from members.models import Member

# Uguaglianza
Member.objects.filter(attivo=True)
Member.objects.filter(nome='Mario')

# Confronto
Member.objects.filter(età__gt=18)     # Maggiore di
Member.objects.filter(età__gte=18)    # Maggiore o uguale
Member.objects.filter(età__lt=65)     # Minore di
Member.objects.filter(età__lte=65)    # Minore o uguale

    Lookup su stringhe

    
# Contiene (case-sensitive)
Member.objects.filter(nome__contains='ari')

# Contiene (case-insensitive)
Member.objects.filter(nome__icontains='mario')

# Inizia con
Member.objects.filter(cognome__startswith='R')

# Finisce con
Member.objects.filter(email__endswith='@gmail.com')

    Filtri su date

    
from datetime import date

Member.objects.filter(data_iscrizione__year=2024)
Member.objects.filter(data_iscrizione__month=6)
Member.objects.filter(data_iscrizione__gt=date(2024, 1, 1))

    Filtri combinati

    
# AND (virgola = AND)
Member.objects.filter(attivo=True, nome__startswith='M')

# OR (richiede Q objects)
from django.db.models import Q
Member.objects.filter(Q(nome='Mario') | Q(nome='Luigi'))

# NOT
Member.objects.filter(~Q(attivo=True))
# Equivalente a:
Member.objects.exclude(attivo=True)

    Aggiungi Campo Slug

    Aggiungere un Campo Slug

    Lo **slug** è una versione URL-friendly di un testo (es: "mio-articolo").

    Aggiungi slug al model

    
# members/models.py
from django.db import models
from django.utils.text import slugify

class Articolo(models.Model):
    titolo = models.CharField(max_length=200)
    slug = models.SlugField(unique=True, max_length=200)
    contenuto = models.TextField()
    data_pubblicazione = models.DateTimeField(auto_now_add=True)

    def save(self, *args, **kwargs):
        if not self.slug:
            self.slug = slugify(self.titolo)
        super().save(*args, **kwargs)

    def __str__(self):
        return self.titolo

    def get_absolute_url(self):
        from django.urls import reverse
        return reverse('articolo-dettaglio', kwargs={'slug': self.slug})

    Slug unico per duplicati

    
import uuid

def save(self, *args, **kwargs):
    if not self.slug:
        base_slug = slugify(self.titolo)
        slug = base_slug
        while Articolo.objects.filter(slug=slug).exists():
            slug = f"{base_slug}-{uuid.uuid4().hex[:8]}"
        self.slug = slug
    super().save(*args, **kwargs)

    I field lookups si usano nei metodi `filter()`, `exclude()`, `get()`.

    Lookup di confronto

    
# Uguaglianza (default)
.filter(nome='Mario')
.filter(nome__exact='Mario')      # Identico, case-sensitive
.filter(nome__iexact='mario')     # Case-insensitive

# Numeri
.filter(età__gt=18)               # Maggiore di (>)
.filter(età__gte=18)              # Maggiore o uguale (>=)
.filter(età__lt=65)               # Minore di (<)
.filter(età__lte=65)              # Minore o uguale (<=)

    Lookup su stringhe

    
.filter(email__contains='gmail')      # Contiene
.filter(email__icontains='gmail')     # Contiene (case-insens.)
.filter(nome__startswith='Mar')       # Inizia con
.filter(nome__istartswith='mar')      # Inizia (case-insens.)
.filter(email__endswith='.com')       # Finisce con
.filter(email__iendswith='.com')      # Finisce (case-insens.)
.filter(nome__regex=r'^[A-Z]')        # Regex
.filter(nome__iregex=r'^[a-z]')       # Regex case-insensitive

    Lookup su date

    
.filter(data__date=date(2024, 6, 15))  # Data esatta
.filter(data__year=2024)               # Anno
.filter(data__month=6)                 # Mese
.filter(data__day=15)                  # Giorno
.filter(data__week=25)                 # Settimana
.filter(data__week_day=2)              # Giorno settimana
.filter(data__quarter=2)               # Trimestre
.filter(data__time=time(14, 30))       # Orario
.filter(data__hour=14)                 # Ora
.filter(data__minute=30)              # Minuto
