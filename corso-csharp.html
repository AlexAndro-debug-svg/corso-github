Introduzione a C# e .NET

C# (si pronuncia “C sharp”) è un linguaggio moderno, orientato agli oggetti, creato da Microsoft nel 2000. Gira sulla piattaforma .NET, che fornisce un'enorme libreria e permette di sviluppare applicazioni Windows, web (ASP.NET), mobile (MAUI) e videogiochi con il motore Unity.

C# unisce la potenza di C++ alla semplicità e sicurezza di linguaggi come Java: è fortemente tipizzato, ha la garbage collection automatica (gestione della memoria) ed è uno dei linguaggi più richiesti nel mondo del lavoro.

📌 C# non si esegue nel browser: serve il .NET SDK. In questo corso gli esempi si leggono e si modificano con la sintassi colorata.

Programmazione a oggetti (OOP)

C# è interamente orientato agli oggetti. Tutto è organizzato in classi e oggetti; i pilastri sono incapsulamento, ereditarietà, polimorfismo e astrazione.

📦 Una classe è lo “stampo”, gli oggetti sono i prodotti creati da quello stampo.

Classi e oggetti

Una classe definisce attributi e metodi. Con new si crea un oggetto.

Membri di classe

I membri di una classe sono i campi (variabili), i metodi (funzioni) e le proprietà. I membri static appartengono alla classe, non alle singole istanze.

Astrazione (classi astratte)

L'astrazione nasconde i dettagli implementativi ed espone solo l'essenziale. Una abstract class non può essere istanziata e può contenere metodi astratti (senza corpo) che le classi derivate devono implementare.

Esempio
📌 Una classe astratta può avere sia metodi astratti (da implementare) sia metodi concreti (già implementati). Non può essere istanziata direttamente.

Le interfacce

Un'interfaccia definisce un contratto di metodi che le classi devono implementare. Per convenzione il nome inizia con la “I”.

Quiz C# — verifica le tue conoscenze

  1. Cos'è il .NET?
    Risposta: Una piattaforma di sviluppo open source di Microsoft che esegue C#. Comprende il runtime (CLR), le librerie di base e gli strumenti di compilazione.
  2. Differenza tra == ed .Equals() per le stringhe?
    Risposta: Per le stringhe in C# entrambi confrontano il contenuto. Per gli oggetti, == confronta i riferimenti se non è sovrascritto.
  3. Cosa sono le proprietà (properties)?
    Risposta: Membri che combinano un campo con i metodi get e set, permettendo accesso controllato con sintassi da campo.
  4. Differenza tra classe astratta e interfaccia?
    Risposta: La classe astratta può avere implementazioni e stato; l'interfaccia definisce solo contratti. Una classe eredita da una sola classe ma implementa più interfacce.
  5. Cos'è il garbage collector?
    Risposta: Il componente del CLR che libera automaticamente la memoria degli oggetti non più referenziati, evitando i memory leak.

Domande da colloquio C#

  • 💬 Differenza tra class e struct? — Le classi sono reference type (allocate sull'heap); le struct sono value type (sullo stack). Le struct si copiano per valore.
  • 💬 Cos'è il boxing/unboxing? — Boxing converte un value type in object (heap); unboxing fa il contrario. Operazioni costose da evitare nei loop.
  • 💬 Cosa sono i delegate? — Tipi che rappresentano riferimenti a metodi, base per eventi e callback (simili ai puntatori a funzione).
  • 💬 Cos'è LINQ? — Language Integrated Query: permette di interrogare collezioni, database e XML con sintassi fluente e leggibile.
  • 💬 Differenza tra IEnumerable e List? — IEnumerable è l'interfaccia base per l'iterazione (lazy); List è una collezione concreta con accesso casuale.
  • 💬 Cosa fa async/await? — Permette la programmazione asincrona non bloccante, fondamentale per operazioni I/O e web.
