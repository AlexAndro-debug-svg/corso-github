Introduzione a React

React è una libreria JavaScript open-source sviluppata da Facebook (Meta) per costruire interfacce utente interattive. È la tecnologia frontend più usata al mondo ed è alla base di app come Facebook, Instagram, WhatsApp Web e migliaia di applicazioni aziendali.

React introduce un approccio basato su componenti riutilizzabili: ogni pezzo dell'interfaccia (pulsante, menu, modulo) è un componente indipendente con la propria logica. Questo rende il codice più organizzato, testabile e scalabile.

Perché imparare React?

  • 🏆 La libreria frontend più richiesta nelle offerte di lavoro
  • ⚡ Virtual DOM per aggiornamenti ultra-veloci dell'interfaccia
  • 🧩 Architettura a componenti riutilizzabili
  • 🌍 Ecosistema enorme: React Router, Redux, Next.js, React Native
  • 📱 Con React Native puoi creare app mobile iOS e Android
⚛️ React non è un framework completo: gestisce solo la view (la UI). Per routing, state globale e altro si usano librerie aggiuntive dell'ecosistema React.

Render HTML

React renderizza i componenti nel DOM reale attraverso il Virtual DOM: una rappresentazione in memoria dell'interfaccia. Quando lo stato cambia, React calcola solo le differenze (diff) e aggiorna solo ciò che è necessario, rendendo le app molto performanti.

Upgrade — React 18

React 18 (2022) ha introdotto importanti novità rispetto alle versioni precedenti. La più visibile è la nuova API createRoot che sostituisce il vecchio ReactDOM.render.

React 17 (vecchio) vs React 18 (nuovo)

Novità di React 18

  • Concurrent Mode: React può interrompere e riprendere il rendering per mantenere l'app reattiva
  • Automatic Batching: gli aggiornamenti di stato multipli vengono raggruppati automaticamente
  • useTransition e useDeferredValue: hook per gestire transizioni UI non urgenti
  • Suspense migliorato: supporto per il data fetching asincrono

ES6 in React

React fa ampio uso delle funzionalità moderne di JavaScript (ES6+). Conoscerle è fondamentale per scrivere codice React idiomatico.

Arrow functions, destructuring, spread
💡 Lo spread operator è molto usato in React per aggiornare lo state in modo immutabile senza modificare l'oggetto originale.

JSX — Introduzione

JSX (JavaScript XML) è un'estensione della sintassi JavaScript che permette di scrivere markup simile all'HTML direttamente nel codice JS. Babel lo compila in chiamate React.createElement().

JSX vs createElement

Regole fondamentali JSX

  • Ogni componente deve restituire un solo elemento radice (usa un <div> o un Fragment <>...</>)
  • I tag vanno sempre chiusi: <br />, <img />
  • Usa className invece di class
  • Usa htmlFor invece di for

JSX — Espressioni

In JSX puoi inserire qualsiasi espressione JavaScript tra parentesi graffe {}. Puoi usare variabili, operazioni matematiche, chiamate a funzioni e operatori ternari.

Espressioni JSX
📌 Le parentesi graffe accettano solo espressioni, non istruzioni (if, for, ecc.). Per la logica usa operatori ternari o funzioni esterne.

JSX — Attributi

Gli attributi JSX seguono la convenzione camelCase. Molti attributi HTML hanno nomi diversi in JSX per evitare conflitti con parole riservate JavaScript.

Componenti

I componenti sono i mattoncini di React. Ogni componente è una funzione (o classe) che riceve dati in input (props) e restituisce JSX. I componenti si compongono tra loro per costruire interfacce complesse.

Componenti funzionali
💡 I nomi dei componenti devono iniziare con la lettera maiuscola: React usa questa convenzione per distinguere i componenti custom dai tag HTML nativi.

Class Components

Prima degli hook (React 16.8), i componenti con stato dovevano essere scritti come classi. Oggi si preferiscono i componenti funzionali con hook, ma è importante conoscere le classi per lavorare su codice legacy.

Class Component

Props

Le props (proprietà) sono il meccanismo con cui un componente genitore passa dati a un componente figlio. Le props sono immutabili: il figlio può leggerle ma non modificarle.

Passare e ricevere props
📌 Le props fluiscono sempre dall'alto verso il basso (dal genitore al figlio). Per passare dati in senso contrario si usano le callback.

Props — Destructuring

Il destructuring ES6 rende il codice più pulito: invece di scrivere props.nome, puoi estrarre le proprietà direttamente nei parametri della funzione.

Destructuring delle props
💡 Puoi definire valori di default direttamente nel destructuring: { colore = "blu" }.

Props — Children

La prop speciale children contiene tutto ciò che viene scritto tra i tag di apertura e chiusura del componente. Permette di creare componenti "contenitore" generici.

Props children

Rendering condizionale

React permette di mostrare o nascondere elementi in base a condizioni. Le tecniche principali sono l'operatore ternario, l'operatore && e le variabili JSX.

Liste

Form

In React i form si gestiscono con i componenti controllati: il valore dell'input è legato allo state, e ogni modifica aggiorna lo state tramite onChange. React diventa così la "fonte di verità" per i dati del form.

Portals

I React Portals permettono di renderizzare un componente figlio in un nodo DOM diverso dal genitore. Sono utili per modali, tooltip e dropdown che devono "uscire" dal flusso DOM normale.

ReactDOM.createPortal
📌 In index.html aggiungi <div id="modal-root"></div> accanto a <div id="root">.

Suspense

React.Suspense permette di mostrare un contenuto di fallback (es. uno spinner) mentre un componente figlio viene caricato in modo asincrono. È usato con React.lazy() per il code splitting.

React.lazy e Suspense
💡 Suspense migliora le performance dell'app: i bundle vengono scaricati solo quando l'utente naviga verso quella sezione (code splitting).

CSS Modules

I CSS Modules sono file CSS normali con l'estensione .module.css. Vite e Create React App li supportano nativamente. I nomi delle classi vengono trasformati in identificatori univoci, eliminando i conflitti.

Bottone.module.css + Bottone.jsx
📌 Vite trasforma styles.bottone in un nome univoco come Bottone_bottone__3xKj1, garantendo l'isolamento degli stili.

CSS-in-JS

Le librerie CSS-in-JS come styled-components e Emotion permettono di scrivere CSS direttamente nel file JavaScript, con pieno supporto a props dinamiche, temi e pseudo-classi.

styled-components

React Router

React Router è la libreria standard per il routing nelle app React. Permette di navigare tra "pagine" senza ricaricare il browser (Single Page Application).

React Router v6 — Setup base
💡 Usa useNavigate() per navigare via codice e useParams() per leggere i parametri dinamici dell'URL (/prodotti/:id).

Transitions — useTransition

L'hook useTransition (React 18) permette di marcare alcuni aggiornamenti di stato come "non urgenti". React dà priorità agli aggiornamenti urgenti (input utente) e posticipa quelli secondari, mantenendo l'interfaccia reattiva.

useTransition

Forward Ref

React.forwardRef permette a un componente figlio di esporre un riferimento DOM al genitore. Utile per componenti di input personalizzati che devono essere controllati dall'esterno.

forwardRef

Higher-Order Components (HOC)

Un HOC è una funzione che riceve un componente e ne restituisce uno nuovo con funzionalità aggiuntive. È un pattern per il riuso della logica tra componenti (oggi spesso sostituito dagli hook).

HOC — withAuth

Sass in React

Sass è un pre-processore CSS che aggiunge variabili, nesting, mixin e funzioni al CSS. In Vite basta installare il pacchetto sass e usare l'estensione .scss.

Hook — useState

useState è l'hook più fondamentale: permette di aggiungere stato locale a un componente funzionale. Restituisce il valore corrente e una funzione per aggiornarlo.

useState — Contatore
💡 Quando il nuovo stato dipende da quello precedente, usa la forma funzionale: setCount(c => c + 1). Evita race condition in aggiornamenti multipli.

Hook — useEffect

useEffect gestisce gli effetti collaterali: fetch di dati, sottoscrizioni a eventi, aggiornamento del DOM. Sostituisce componentDidMount, componentDidUpdate e componentWillUnmount.

useEffect — fetch dati
Array dipendenzeComportamento
OmessoEseguito dopo ogni render
[] vuotoSolo al mount (una volta)
[val]Quando val cambia

Hook — useContext

useContext consente di condividere dati tra componenti senza passare props a ogni livello (prop drilling). Il Context è ideale per temi, lingua, utente autenticato.

useContext — Tema

Hook — useRef

useRef restituisce un oggetto mutabile con una proprietà .current. È usato per accedere direttamente agli elementi DOM e per conservare valori che non devono causare un re-render quando cambiano.

useRef — DOM e timer

Hook — useReducer

useReducer è un'alternativa a useState per stato complesso. Si ispira al pattern Redux: un reducer riceve lo stato corrente e un'azione, e restituisce il nuovo stato.

useReducer — Contatore avanzato
💡 Preferisci useReducer quando hai molte variabili di stato correlate o logica di transizione complessa.

Hook — useCallback

useCallback restituisce una versione memoizzata di una funzione callback. È utile per ottimizzare le performance quando si passa una callback a componenti figli profondi: evita che il figlio si re-renderizzi inutilmente ad ogni render del genitore.

useCallback

Hook — useMemo

useMemo memoizza il risultato di un calcolo costoso: lo ricalcola solo quando le dipendenze cambiano. È utile per filtrare/ordinare grandi liste o calcoli pesanti.

useMemo — Filtro lista

Custom Hooks

I Custom Hook sono funzioni JavaScript che iniziano con use e possono chiamare altri hook. Permettono di estrarre e riutilizzare logica stateful tra più componenti.

Custom Hook — useFetch
💡 I Custom Hook sono il modo moderno per condividere logica in React. Seguono sempre la convenzione use all'inizio del nome.

Quiz React

Metti alla prova le tue conoscenze con queste domande:

  1. ❓ Cosa restituisce useState(0)?
    R: Un array con il valore corrente (0) e la funzione di aggiornamento: [0, setStato].
  2. ❓ Qual è la differenza tra useCallback e useMemo?
    R: useCallback memoizza una funzione; useMemo memoizza il risultato di un calcolo.
  3. ❓ Cosa succede se non si fornisce una key in una lista React?
    R: React non riesce a tracciare efficientemente gli elementi → warning in console e possibili bug di rendering.
  4. ❓ Quando viene eseguito il cleanup di useEffect?
    R: Prima che il componente venga smontato e prima che l'effetto venga rieseguito (se le dipendenze cambiano).
  5. ❓ Cos'è il "prop drilling"?
    R: Passare props attraverso molti livelli di componenti annidati. Si risolve con Context o Redux.
  6. ❓ Quali hook sono disponibili solo in componenti funzionali?
    R: Tutti gli hook React (useState, useEffect, ecc.) — non si possono usare nelle classi.

Domande da colloquio React

  • 💬 Differenza tra stato locale e stato globale? — Lo stato locale (useState) è isolato nel componente. Lo stato globale (Context, Redux, Zustand) è condiviso tra più componenti.
  • 💬 Cos'è il Virtual DOM e perché è importante? — Una rappresentazione in memoria del DOM reale. React calcola le differenze (reconciliation) e aggiorna solo le parti cambiate, rendendo le app molto veloci.
  • 💬 Quando usare useEffect? — Per effetti collaterali: fetch dati, sottoscrizioni, interazioni con API esterne, aggiornamenti del titolo della pagina.
  • 💬 Cosa sono le Controlled Components? — Componenti form il cui valore è gestito dallo stato React, non dal DOM. React è la fonte di verità per il valore.
  • 💬 Perché le key nelle liste devono essere univoche? — React le usa per riconoscere quali elementi sono cambiati, aggiunti o rimossi, ottimizzando il re-rendering.
  • 💬 Differenza tra useCallback e useMemo? — useCallback memoizza una funzione; useMemo memoizza il risultato di un calcolo. Entrambi evitano rielaborazioni inutili.
  • 💬 Cos'è il Context e quando usarlo? — Permette di passare dati in profondità senza prop drilling. Adatto per temi, lingua, autenticazione. Non è sostituto di Redux per stato complesso e frequentemente aggiornato.
