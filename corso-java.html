Introduzione a Java

Java è un linguaggio di programmazione orientato agli oggetti, creato da Sun Microsystems nel 1995 e oggi mantenuto da Oracle. È uno dei linguaggi più usati al mondo per applicazioni aziendali, app Android, sistemi bancari e software di grandi dimensioni.

Il suo motto è “Write Once, Run Anywhere”: il codice Java viene compilato in bytecode ed eseguito dalla JVM (Java Virtual Machine), perciò gira su qualsiasi sistema operativo senza modifiche. Java è fortemente tipizzato e segue rigorosamente il paradigma a oggetti.

📌 Java non si esegue nel browser: in questo corso gli esempi si leggono e si modificano con la sintassi colorata. Per eseguirli ti serve il JDK installato sul tuo computer (vedi “Installazione”).

Programmazione a oggetti (OOP)

Java è un linguaggio profondamente orientato agli oggetti. Tutto ruota attorno a classi (i modelli) e oggetti (le istanze). I pilastri sono incapsulamento, ereditarietà, polimorfismo e astrazione: permettono di scrivere software grande, ordinato e riutilizzabile.

📦 Una classe è lo “stampo”, gli oggetti sono i “biscotti” creati con quello stampo.

Classi e oggetti

Una classe definisce attributi (dati) e metodi (comportamenti). Con new si crea un oggetto.

Esempio

Gli attributi

Gli attributi (o campi) sono le variabili che descrivono lo stato di un oggetto. Ogni oggetto ha la propria copia.

Esempio

I metodi di classe

I metodi definiscono il comportamento degli oggetti e possono usare i loro attributi.

Esempio

I costruttori

Un costruttore è un metodo speciale (stesso nome della classe) chiamato automaticamente con new per inizializzare l'oggetto. La parola chiave this indica l'oggetto corrente.

Esempio

I modificatori d'accesso

I modificatori d'accesso stabiliscono la visibilità di attributi e metodi: public (ovunque), private (solo nella classe), protected (classe e sottoclassi), o di default (stesso package).

ModificatoreVisibilità
publicDa qualsiasi classe
privateSolo dentro la classe
protectedClasse, package e sottoclassi

L'incapsulamento

L'incapsulamento protegge i dati rendendo gli attributi private e fornendo metodi pubblici getter e setter per leggerli e modificarli in modo controllato.

Esempio

L'ereditarietà

Con extends una classe (figlia) eredita attributi e metodi da un'altra (genitore), riutilizzando il codice. super richiama il genitore.

Esempio

Il polimorfismo

Il polimorfismo permette di usare lo stesso metodo con comportamenti diversi a seconda dell'oggetto. Con l'override una sottoclasse ridefinisce un metodo del genitore.

Esempio

Le interfacce

Un'interfaccia definisce un “contratto” di metodi che le classi devono implementare con implements. È il modo di Java per l'astrazione e per ottenere una forma di ereditarietà multipla.

Esempio

ArrayList e collezioni

L'ArrayList è una lista dinamica (a dimensione variabile), parte del Collections Framework. A differenza degli array può crescere e ridursi.

Le eccezioni sono errori che si verificano durante l'esecuzione. Si gestiscono con try ... catch (e l'opzionale finally) per evitare che il programma vada in crash.

Esempio

Booleans in Java

Un tipo booleano (boolean) può assumere solo due valori: true o false. È il tipo restituito da tutte le espressioni di confronto.

Overloading dei metodi

Esempio

Scope delle variabili

Lo scope (ambito) di una variabile determina dove può essere usata. In Java le variabili dichiarate dentro un blocco { } esistono solo all'interno di quel blocco.

Esempio
  • Variabili locali: esistono solo nel metodo/blocco in cui sono dichiarate
  • Attributi di classe: visibili in tutti i metodi della classe
  • Variabili statiche: condivise tra tutte le istanze della classe

Ricorsione

Un metodo ricorsivo è un metodo che chiama se stesso. Ogni chiamata ricorsiva deve avvicinarsi a un caso base che interrompe la ricorsione.

Fattoriale ricorsivo
⚠️ Senza caso base la ricorsione è infinita e genera StackOverflowError. Usare la ricorsione con attenzione per dati molto grandi.

Debugging in Java

Il debugging è il processo di individuazione e correzione degli errori. Java distingue tre tipi di errori: syntax errors (errori di sintassi), runtime errors (errori a runtime) e logic errors (errori logici).

  • Syntax error: rilevato dal compilatore (manca ;, parentesi, ecc.)
  • Runtime error: genera un'eccezione durante l'esecuzione (NullPointerException, ArrayIndexOutOfBoundsException)
  • Logic error: il programma gira ma produce risultati sbagliati

Strumenti di debug: System.out.println() per stampe di controllo, IDE debugger (IntelliJ IDEA, Eclipse, VS Code) per breakpoint, watch variables, step-by-step execution.

💡 Usa il debugger dell'IDE: posiziona un breakpoint su una riga, avvia in modalità debug e osserva i valori delle variabili in tempo reale.

Quiz Java — metti alla prova le tue conoscenze

Rispondi mentalmente a queste domande e poi espandi la risposta per verificare.

  1. Qual è la differenza tra == e .equals() in Java?
    Risposta: == confronta i riferimenti (indirizzi in memoria); .equals() confronta il contenuto. Per le stringhe usa sempre .equals().
  2. Cosa succede se non si chiama super() in un costruttore figlio?
    Risposta: Java lo chiama automaticamente se la superclasse ha un costruttore senza argomenti. Altrimenti è necessario chiamarlo esplicitamente.
  3. Qual è la differenza tra ArrayList e LinkedList?
    Risposta: ArrayList usa un array dinamico (accesso casuale O(1), inserimento in mezzo O(n)); LinkedList usa nodi collegati (inserimento in testa/coda O(1), accesso casuale O(n)).
  4. Cosa significa static?
    Risposta: Un membro static appartiene alla classe, non alle singole istanze. È condiviso tra tutti gli oggetti e si accede tramite il nome della classe.
  5. Quando si usa un'interfaccia invece di una classe astratta?
    Risposta: Si usa l'interfaccia quando si vogliono definire comportamenti da condividere tra classi non correlate. Una classe può implementare più interfacce ma estendere solo una classe.

Domande da colloquio Java

Le domande Java più frequenti in colloqui tecnici e selezioni universitarie:

  • 💬 Spiega il principio SOLID. — Cinque principi di buona progettazione OOP: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
  • 💬 Cosa è il garbage collector? — Processo JVM che libera automaticamente la memoria non più referenziata.
  • 💬 Differenza tra HashMap e Hashtable? — HashMap non è sincronizzata (più veloce), Hashtable è thread-safe ma obsoleta. Per concorrenza usa ConcurrentHashMap.
  • 💬 Cos'è il pattern Singleton? — Garantisce che una classe abbia una sola istanza. Si implementa con costruttore privato e metodo statico getInstance().
  • 💬 Cosa sono le Stream API? — Introdotte in Java 8, permettono operazioni funzionali (filter, map, reduce) su collezioni in modo fluente e leggibile.
  • 💬 Differenza tra abstract class e interface? — La classe astratta può avere stato e implementazioni parziali; l'interfaccia definisce solo contratti (da Java 8 può avere metodi default). Una classe estende una sola classe ma implementa più interfacce.

Riepilogo Java

I concetti fondamentali che ogni programmatore Java deve padroneggiare:

  • ✅ Sintassi: tipi primitivi, variabili, operatori, casting, stringhe
  • ✅ Controllo: if/else, switch, while, for, break, continue
  • ✅ Metodi: parametri, return, overloading, scope, ricorsione
  • ✅ OOP: classi, costruttori, incapsulamento, ereditarietà, polimorfismo, interfacce
  • ✅ OOP avanzato: abstract, enum, inner classes, anonymous, this, super, package
  • ✅ Eccezioni: try/catch/finally, multi-catch, eccezioni custom, try-with-resources
  • ✅ File I/O: File, FileWriter, BufferedReader, FileInputStream
  • ✅ Collections: ArrayList, LinkedList, HashSet, TreeSet, HashMap, TreeMap, Iterator
  • ✅ Java moderno: Generics, lambda, Stream API, thread, regex, annotazioni
📌 Java è il linguaggio più richiesto nelle aziende italiane per backend enterprise. Padroneggiarlo apre le porte a ruoli da Java Developer, Android Developer e Software Engineer.
