Introduzione a TypeScript

TypeScript è un linguaggio di programmazione open-source sviluppato da Microsoft che estende JavaScript aggiungendo la tipizzazione statica opzionale. Il codice TypeScript viene compilato (transpilato) in JavaScript puro, quindi gira ovunque gira JavaScript: browser, Node.js, Deno.

TypeScript non sostituisce JavaScript: lo potenzia. Ogni file JavaScript valido è già TypeScript valido. Puoi adottarlo gradualmente nel tuo progetto esistente.

Perché usare TypeScript?

  • 🐛 Errori a compile-time: individua i bug prima di eseguire il codice
  • 🧠 Autocompletamento intelligente: l'editor conosce i tipi e suggerisce i metodi corretti
  • 📖 Codice auto-documentato: i tipi spiegano cosa si aspetta ogni funzione
  • 🔄 Refactoring sicuro: rinominare un campo aggiorna tutto il progetto
  • 🏢 Standard industriale: usato da Google, Microsoft, Airbnb, Slack, VS Code stesso
📘 TypeScript non gira direttamente nel browser: viene compilato in JavaScript con il comando tsc. Usa il TypeScript Playground per provare subito senza installare nulla.
JS vs TS — stesso errore, esiti diversi

Get Started — Installazione

Per iniziare con TypeScript ti serve Node.js installato. TypeScript si installa globalmente via npm e si usa con il compilatore tsc.

Installazione e primo progetto

Avvio rapido senza installazione

Per sperimentare subito senza installare nulla, usa il TypeScript Playground ufficiale su typescriptlang.org/play: scrivi TypeScript a sinistra e vedi il JavaScript compilato a destra in tempo reale.

Tipi semplici

TypeScript supporta tutti i tipi primitivi di JavaScript, con la possibilità di dichiararli esplicitamente. I principali tipi primitivi sono string, number, boolean.

Tipi primitivi

Dichiarazione esplicita e inferenza

TypeScript può dedurre (inferire) il tipo automaticamente dal valore assegnato. Non è sempre necessario scrivere il tipo esplicitamente: TypeScript lo capisce dal contesto.

Esplicito vs Inferito
💡 Regola pratica: lascia che TypeScript inferisca quando inizializzi con un valore. Dichiara esplicitamente quando la variabile è dichiarata senza valore o il tipo non è ovvio.

Tipi speciali

TypeScript ha alcuni tipi speciali che non esistono (o hanno significato diverso) in JavaScript puro.

Array

Gli array TypeScript sono tipizzati: ogni elemento deve rispettare il tipo dichiarato. Ci sono due sintassi equivalenti per dichiarare un array tipizzato.

Array tipizzati

Tuple

Una tuple è un array con un numero fisso di elementi, ognuno con il proprio tipo. A differenza degli array normali, l'ordine e il tipo di ogni posizione sono definiti con precisione.

Tuple TypeScript

Object Types

TypeScript permette di descrivere la struttura degli oggetti definendo il tipo di ogni proprietà. Puoi usare la sintassi inline oppure un type alias o un'interfaccia.

Tipi degli oggetti

Enums

Un enum è un insieme di costanti con nome. Rendono il codice più leggibile rispetto ai valori magici come 0, 1, 2.

Enum numerici e a stringa

Union Types

Un union type permette a una variabile di avere uno tra più tipi possibili, separati dall'operatore |.

Funzioni

TypeScript permette di tipizzare i parametri e il valore di ritorno delle funzioni, rendendo l'API di ogni funzione chiara e verificata a compile-time.

Funzioni tipizzate

Casting (Type Assertions)

Il casting in TypeScript permette di dire al compilatore "fidati, so il tipo di questo valore". Non converte il valore a runtime, ma dice al type checker come trattarlo.

Type assertions
⚠️ Il casting non converte il valore a runtime: se asserisci il tipo sbagliato, l'errore comparirà comunque durante l'esecuzione. Usare con cura.

Classi

TypeScript estende le classi ES6 con modificatori di accesso, proprietà tipizzate, classi astratte e molto altro.

Classi TypeScript

Generics base

I generics permettono di scrivere codice riutilizzabile che funziona con qualsiasi tipo, mantenendo la type-safety. Invece di usare any, usi un tipo parametro come T.

Funzioni e classi generiche

Utility Types

TypeScript include molti utility types predefiniti che trasformano un tipo esistente in uno nuovo. Evitano di riscrivere strutture simili da zero.

Utility Types più usati

Keyof

L'operatore keyof restituisce un union type con tutte le chiavi di un tipo oggetto. È molto utile per creare funzioni generiche che operano sulle proprietà di un oggetto.

keyof operator

Null e Undefined

Con strictNullChecks attivo (default in TypeScript moderno), null e undefined non sono assegnabili a qualsiasi tipo: devono essere esplicitamente dichiarati.

Gestione di null e undefined

Definitely Typed (@types)

Molte librerie JavaScript non includono i tipi TypeScript. Il progetto Definitely Typed fornisce i file di dichiarazione dei tipi (.d.ts) per migliaia di librerie populari, sotto il namespace @types.

Installare i tipi di una libreria
💡 Molte librerie moderne includono già i tipi (React, Axios, Zod, ecc.) — non serve installare @types separatamente. Controlla se esiste già un campo "types" in package.json.

Configurazione — tsconfig.json

Il file tsconfig.json controlla il comportamento del compilatore TypeScript per il progetto. È la prima cosa da configurare in ogni progetto TS.

tsconfig.json tipico

TypeScript con Node.js

TypeScript si integra perfettamente con Node.js. Per lo sviluppo locale puoi usare ts-node (esecuzione diretta) o tsx (più veloce). Per la produzione si compila con tsc.

Tipi avanzati

TypeScript offre operatori per costruire tipi complessi a partire da tipi esistenti: intersection, template literal types e molto altro.

Intersection, template literals, infer

Type Guards

I type guards sono controlli a runtime che permettono a TypeScript di restringere (narrowing) il tipo di una variabile all'interno di un blocco condizionale.

Type guards comuni

Tipi condizionali

I tipi condizionali permettono di scegliere un tipo in base a una condizione, usando la sintassi T extends U ? X : Y.

Conditional Types

Mapped Types

I mapped types creano un nuovo tipo iterando sulle chiavi di un tipo esistente. Sono la base su cui sono costruiti gli utility types come Partial, Readonly, Required.

Mapped Types

Inferenza dei tipi avanzata

TypeScript inferisce i tipi in molti contesti avanzati: valori di ritorno, parametri di callback, variabili distruttturate e altro ancora.

Inferenza avanzata

Literal Types

I literal types permettono di specificare non solo il tipo ma il valore esatto che una variabile può assumere. Sono molto usati per creare API espressive e sicure.

Literal Types

Namespaces

I namespaces (ex "moduli interni") organizzano il codice in scope nominati per evitare collisioni di nomi. Nei progetti moderni con ES Modules sono meno usati, ma restano utili per librerie e file di dichiarazione.

Namespaces

Index Signatures

Le index signatures permettono di definire il tipo dei valori di un oggetto quando le chiavi non sono note in anticipo (dizionari, mappe).

Index Signatures

Declaration Merging

TypeScript può unire più dichiarazioni con lo stesso nome in un'unica definizione. Funziona con interfacce, namespace e, in alcuni casi, funzioni.

Declaration Merging

Async Programming

TypeScript si integra perfettamente con Promise, async/await e i tipi asincroni. Tipizzare le Promise rende il codice asincrono più sicuro.

Promise e async/await tipizzati

Decorators

I decorators sono funzioni speciali che possono annotare e modificare classi, metodi, proprietà e parametri. Sono standard ECMAScript dalla versione TypeScript 5.

Decorators di classe e metodo

TypeScript in progetti JavaScript

Puoi usare TypeScript anche senza rinominare i file in .ts: attiva checkJs nel tsconfig e TypeScript controllerà i tuoi file JS esistenti con i commenti JSDoc.

TS in file .js con JSDoc

Migrazione da JavaScript a TypeScript

Migrare un progetto JS esistente a TypeScript è un processo graduale. Non è necessario convertire tutto in una volta.

Strategia di migrazione graduale
💡 Strategia consigliata: inizia abilitando TypeScript solo sui nuovi file, poi converti gradualmente i file esistenti partendo dai più semplici (tipi/utility) verso i più complessi (componenti principali).

Error Handling

TypeScript migliora la gestione degli errori tipizzando le eccezioni. Dal TS 4.4, puoi usare useUnknownInCatchVariables per forzare il controllo del tipo negli errori catturati.

Best Practices TypeScript

Seguire le convenzioni della comunità TypeScript rende il codice più leggibile, manutenibile e sicuro.

Buone pratiche
  • 🔒 Abilita sempre "strict": true nei nuovi progetti
  • 🚫 Evita any — usa unknown e i type guard
  • 📐 Preferisci interface per le API pubbliche e type per le union
  • 🔑 Usa i discriminated union per gli stati complessi
  • 🛠️ Configura ESLint con @typescript-eslint
  • 📦 Organizza i tipi in file dedicati (types/ o *.types.ts)

Quiz TypeScript

  1. ❓ Qual è la differenza tra any e unknown?
    R: any disabilita completamente il type checking; unknown richiede un type guard prima dell'uso — è la versione sicura di any.
  2. ❓ Quando preferire interface a type?
    R: interface quando definisci la forma di un oggetto o una classe (supporta extends e merging); type per union, intersection e tipi più complessi.
  3. ❓ Cosa fa Partial<T>?
    R: Rende tutte le proprietà di T opzionali (?). Utile per oggetti di aggiornamento parziale.
  4. ❓ Cos'è un type guard?
    R: Un controllo a runtime (typeof, instanceof, in) che permette a TypeScript di restringere il tipo di una variabile in un blocco condizionale.
  5. ❓ Cosa significa strict: true nel tsconfig?
    R: Abilita tutti i controlli rigorosi: strictNullChecks, noImplicitAny, strictFunctionTypes, ecc.
  6. ❓ Qual è la differenza tra un type alias e un'interfaccia per un union type?
    R: Solo type supporta i union type (string | number); le interfacce non possono esprimere direttamente un union.

Domande da colloquio TypeScript

  • 💬 Cos'è TypeScript e qual è il suo rapporto con JavaScript? — TypeScript è un superset di JavaScript con tipizzazione statica opzionale. Ogni codice JS è TS valido; TS viene compilato in JS prima dell'esecuzione.
  • 💬 Differenza tra interface e type? — Entrambi descrivono strutture di oggetti. interface supporta declaration merging ed è preferita per le API; type supporta union, intersection e tipi complessi.
  • 💬 Cosa sono i generics e perché usarli? — Permettono di scrivere codice riutilizzabile che mantiene la type-safety con qualsiasi tipo, senza ricorrere ad any.
  • 💬 Cosa fa strict: true? — Abilita strictNullChecks, noImplicitAny, strictFunctionTypes e altri controlli rigorosi. Fortemente consigliato in tutti i nuovi progetti.
  • 💬 Differenza tra unknown e any? — any disabilita il type checking; unknown è sicuro perché richiede un type guard prima di operare sul valore.
  • 💬 Cosa sono i discriminated union? — Union di oggetti con un campo discriminante comune (es. tipo: "A" | "B") che permette a TypeScript di narroware il tipo automaticamente con switch/if.
  • 💬 Come tipizzare le props in React con TypeScript? — Con un'interfaccia o type alias passato come generic al componente o usato nel destructuring dei parametri della funzione.
