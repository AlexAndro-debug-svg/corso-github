Introduzione a MySQL

MySQL è uno dei sistemi di gestione di database (DBMS) relazionali più diffusi al mondo, gratuito e open source. È il cuore di moltissimi siti web e applicazioni: è alla base dello stack LAMP (Linux, Apache, MySQL, PHP) e viene gestito spesso tramite l'interfaccia grafica phpMyAdmin.

MySQL usa il linguaggio SQL: tutto ciò che impari qui vale anche per gli altri database relazionali. In questo corso vedremo sia i comandi SQL sia gli aspetti più specifici di MySQL (tipi di dato, AUTO_INCREMENT, funzioni, utenti).

💡 Puoi eseguire le query nel browser: c'è un database di esempio con le tabelle studenti (id, nome, voto, classe, classe_id, citta) e classi (id, aula, classe). Il motore è SQLite, quasi identico a MySQL: dove la sintassi cambia, te lo segnalo.
Esempio

Cos'è un DBMS

Un DBMS (Database Management System) è il software che memorizza, organizza e protegge i dati, permettendo a più utenti e applicazioni di accedervi contemporaneamente in modo sicuro. MySQL è un DBMS relazionale: i dati sono organizzati in tabelle collegate tra loro tramite chiavi.

  • Database: un contenitore di tabelle (es. il database di una scuola).
  • Tabella: una griglia di righe e colonne (es. studenti).
  • Record (riga): una singola voce (es. lo studente Marco).
  • Campo (colonna): una proprietà (es. il voto).

AUTO_INCREMENT

Con AUTO_INCREMENT MySQL genera automaticamente un numero progressivo per la chiave primaria a ogni inserimento: non devi indicare tu l'id.

⚙️ In MySQL: id INT AUTO_INCREMENT PRIMARY KEY. In SQLite (l'editor del corso) si usa INTEGER PRIMARY KEY AUTOINCREMENT: il risultato è lo stesso.

Inserire dati: INSERT INTO

INSERT INTO aggiunge una o più righe. Puoi inserire più record in un solo comando elencando più gruppi di VALUES.

Esempio

Leggere i dati: SELECT

SELECT legge i dati da una tabella. Con * ottieni tutte le colonne, oppure elenchi quelle desiderate. DISTINCT elimina i duplicati.

Esempio

Filtrare: WHERE

La clausola WHERE filtra le righe con una condizione. Puoi combinare più condizioni con AND, OR e NOT.

Esempio

Ordinare e limitare: ORDER BY e LIMIT

ORDER BY ordina i risultati (ASC/DESC); LIMIT restituisce solo le prime N righe. Insieme servono per le classifiche e la paginazione.

Esempio

Modificare: UPDATE

UPDATE modifica righe esistenti. Ricorda sempre il WHERE, altrimenti aggiorni tutte le righe!

Esempio

Cancellare: DELETE

DELETE elimina righe. Con il WHERE selezioni quali; senza, svuoti la tabella.

Le funzioni di aggregazione

Calcolano un valore su più righe: COUNT() conta, SUM() somma, AVG() media, MIN()/MAX() minimo e massimo.

Esempio

LIKE, IN e BETWEEN

LIKE cerca pattern di testo (% = qualsiasi sequenza); IN verifica l'appartenenza a un elenco; BETWEEN seleziona un intervallo.

Esempio

Unire tabelle: JOIN

Il JOIN combina righe di tabelle collegate. Qui uniamo studenti e classi tramite classe_id. INNER JOIN tiene solo le corrispondenze; LEFT JOIN tiene tutte le righe di sinistra.

Esempio

GROUP BY e HAVING

GROUP BY raggruppa le righe per calcolare aggregati per gruppo; HAVING filtra i gruppi (a differenza di WHERE che filtra le righe).

Esempio

Le funzioni stringa

MySQL offre funzioni per manipolare il testo: UPPER(), LOWER(), LENGTH() e CONCAT() per concatenare.

⚙️ In MySQL si concatena con CONCAT(nome, ' - ', citta). Nell'editor del corso (SQLite) si usa l'operatore ||: il risultato è lo stesso.
Esempio

Le funzioni data e ora

MySQL gestisce date e orari con funzioni come NOW() (data e ora attuali), CURDATE() (data odierna) e YEAR(), MONTH(), DAY() per estrarne le parti.

⚙️ In MySQL: SELECT NOW();. Nell'editor (SQLite) l'equivalente è date('now') e datetime('now').
Esempio

Le viste (VIEW)

Una vista è una tabella virtuale basata su una query salvata: la interroghi come una tabella normale. Semplifica le query complesse e le rende riutilizzabili.

Esempio

Gli indici (INDEX)

Un indice accelera le ricerche su una colonna molto usata nei filtri. Migliora le SELECT al prezzo di inserimenti leggermente più lenti.

Esempio

Utenti e privilegi

MySQL gestisce la sicurezza tramite utenti con permessi specifici. Un amministratore crea utenti e assegna loro solo i privilegi necessari (principio del minimo privilegio). Questi comandi si eseguono sul server MySQL reale.

MySQL SQL — cos'è

SQL (Structured Query Language) è il linguaggio con cui si parla a MySQL: serve a creare database e tabelle, inserire dati e interrogarli. MySQL è il gestore (DBMS) più diffuso al mondo per le applicazioni web, SQL è il linguaggio che usa.

💡 In questo corso esegui SQL reale direttamente nel browser su un database di esempio con le tabelle studenti (id, nome, voto, classe, classe_id, citta) e classi (id, aula, classe).

MySQL SELECT DISTINCT

SELECT DISTINCT restituisce solo i valori diversi, eliminando i duplicati.

Esempio

MySQL AND, OR, NOT

Gli operatori logici combinano più condizioni nel WHERE: AND (tutte vere), OR (almeno una), NOT (negazione).

Esempio

MySQL Valori NULL

NULL indica un valore mancante, diverso da 0 o da stringa vuota. Si verifica con IS NULL e IS NOT NULL, mai con =.

SELECT * FROM studenti WHERE citta IS NULL;

MySQL LIMIT

LIMIT restringe il numero di righe restituite. È l'equivalente MySQL del SELECT TOP di SQL Server.

Esempio

MySQL MIN() e MAX()

Le funzioni MIN() e MAX() restituiscono il valore più piccolo e più grande di una colonna.

Esempio

MySQL COUNT(), SUM(), AVG()

COUNT() conta le righe, SUM() somma, AVG() calcola la media.

Esempio

MySQL Wildcards

I caratteri jolly si usano con LIKE: % per qualsiasi sequenza, _ per un singolo carattere.

PatternTrova

MySQL IN

L'operatore IN verifica se un valore è in un elenco, in modo più compatto di tanti OR.

Esempio

MySQL BETWEEN

BETWEEN seleziona i valori in un intervallo (estremi inclusi).

Esempio

MySQL Alias

Gli alias (AS) rinominano temporaneamente una colonna o una tabella, rendendo i risultati più leggibili.

Esempio

MySQL INNER JOIN

L'INNER JOIN restituisce solo le righe che hanno corrispondenza in entrambe le tabelle.

Esempio

MySQL LEFT JOIN

Il LEFT JOIN restituisce tutte le righe della tabella di sinistra e le corrispondenze da destra; dove mancano, i campi sono NULL.

SELECT s.nome, c.aula
FROM studenti s
LEFT JOIN classi c ON s.classe_id = c.id;

MySQL RIGHT JOIN

Il RIGHT JOIN è lo specchio del LEFT: tutte le righe della tabella di destra e le corrispondenze da sinistra.

SELECT s.nome, c.aula
FROM studenti s
RIGHT JOIN classi c ON s.classe_id = c.id;

MySQL CROSS JOIN

Il CROSS JOIN combina ogni riga della prima tabella con ogni riga della seconda (prodotto cartesiano). Da usare con attenzione: il numero di righe può esplodere.

SELECT s.nome, c.aula
FROM studenti s
CROSS JOIN classi c;

MySQL SELF JOIN

Un SELF JOIN unisce una tabella a sé stessa usando gli alias, ad esempio per confrontare righe della stessa tabella.

SELECT a.nome AS studente1, b.nome AS studente2, a.citta
FROM studenti a, studenti b
WHERE a.citta = b.citta AND a.id < b.id;

MySQL UNION e UNION ALL

UNION unisce i risultati di due query eliminando i duplicati; UNION ALL li tiene tutti. Le query devono avere lo stesso numero di colonne.

SELECT nome FROM studenti WHERE classe = '3A'
UNION
SELECT nome FROM studenti WHERE voto >= 8;

MySQL HAVING

HAVING filtra i gruppi creati da GROUP BY (mentre WHERE filtra le righe prima del raggruppamento).

Esempio

MySQL EXISTS

EXISTS verifica se una subquery restituisce almeno una riga, restituendo vero o falso.

SELECT aula FROM classi c
WHERE EXISTS (SELECT 1 FROM studenti s WHERE s.classe_id = c.id);

MySQL ANY e ALL

ANY è vero se almeno un confronto con la subquery è vero; ALL se lo sono tutti.

SELECT nome FROM studenti
WHERE voto > ANY (SELECT voto FROM studenti WHERE classe = '3A');

MySQL INSERT INTO SELECT

INSERT INTO SELECT copia righe da una tabella a un'altra già esistente.

INSERT INTO promossi (nome, voto)
SELECT nome, voto FROM studenti WHERE voto >= 6;

MySQL CASE

L'espressione CASE è l'if/else di SQL: restituisce valori diversi secondo le condizioni.

Esempio

MySQL Funzioni per i NULL

In MySQL IFNULL() e COALESCE() forniscono un valore alternativo quando un campo è NULL.

SELECT nome, IFNULL(citta, 'Non indicata') AS citta
FROM studenti;

MySQL Stored Procedures

Una stored procedure è un blocco SQL salvato nel database e richiamabile per nome, anche con parametri.

DELIMITER //
CREATE PROCEDURE StudentiClasse(IN c VARCHAR(10))
BEGIN
  SELECT * FROM studenti WHERE classe = c;
END //
DELIMITER ;

CALL StudentiClasse('3A');

MySQL Commenti

I commenti documentano le query: -- o # su una riga, /* ... */ su più righe.

# commento MySQL su una riga
SELECT nome FROM studenti;  -- anche così
/* commento
   su più righe */

MySQL DROP DATABASE

DROP DATABASE elimina definitivamente un intero database. Operazione irreversibile.

DROP DATABASE scuola;
⚠️ Prima di un DROP fai sempre un backup del database.

MySQL DROP TABLE

DROP TABLE elimina una tabella con tutti i suoi dati. Per svuotarla mantenendone la struttura si usa invece TRUNCATE TABLE.

DROP TABLE studenti;

MySQL ALTER TABLE

ALTER TABLE modifica la struttura di una tabella esistente: aggiunge, modifica o elimina colonne.

ALTER TABLE studenti ADD email VARCHAR(100);
ALTER TABLE studenti DROP COLUMN email;

MySQL Vincoli (Constraints)

I vincoli definiscono regole sui dati di una colonna per garantirne la correttezza.

VincoloGarantisce
NOT NULLCampo sempre valorizzato
UNIQUEValori tutti diversi
PRIMARY KEYIdentificatore univoco
FOREIGN KEYCollegamento ad altra tabella
CHECKCondizione sui valori
DEFAULTValore predefinito

MySQL NOT NULL

Il vincolo NOT NULL impone che una colonna abbia sempre un valore.

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(50) NOT NULL
);

MySQL UNIQUE

Il vincolo UNIQUE impedisce valori duplicati in una colonna (es. email, matricola).

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  email VARCHAR(100) UNIQUE
);

MySQL PRIMARY KEY

La chiave primaria identifica univocamente ogni riga: è UNIQUE e NOT NULL insieme.

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(50)
);

MySQL FOREIGN KEY

La chiave esterna collega una tabella alla chiave primaria di un'altra, garantendo l'integrità referenziale. È la base delle relazioni in MySQL (motore InnoDB).

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(50),
  classe_id INT,
  FOREIGN KEY (classe_id) REFERENCES classi(id)
);

MySQL CHECK

Il vincolo CHECK limita i valori ammessi in una colonna a quelli che soddisfano una condizione.

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  voto INT CHECK (voto BETWEEN 1 AND 10)
);

MySQL DEFAULT

Il vincolo DEFAULT assegna un valore predefinito quando non se ne fornisce uno.

CREATE TABLE studenti (
  id INT AUTO_INCREMENT PRIMARY KEY,
  citta VARCHAR(50) DEFAULT 'Roma'
);

MySQL SQL Injection (sicurezza)

La SQL Injection è una grave vulnerabilità: inserendo codice SQL nei campi di input, un attaccante può leggere o distruggere dati. Mai concatenare l'input dell'utente nelle query!

-- ❌ PERICOLOSO
"SELECT * FROM utenti WHERE nome = '" + input + "'"
-- input = ' OR '1'='1  → l'attaccante entra senza password
⭐ La difesa sono i prepared statements: l'input è trattato come dato, mai come codice.

MySQL Prepared Statements

I prepared statements separano il codice SQL dai dati con dei segnaposto (?), prevenendo la SQL Injection. È il modo corretto di gestire l'input.

PREPARE stmt FROM 'SELECT * FROM studenti WHERE classe = ?';
SET @c = '3A';
EXECUTE stmt USING @c;
💡 In PHP, Python, Java ecc. il valore dei parametri si passa con apposite funzioni (bindParam, execute([...])), non concatenandolo nella stringa.

Esercizi MySQL con soluzioni (per principianti)

Questi esercizi di MySQL svolti sono pensati per chi vuole fare pratica con query reali. Per ognuno c'è la consegna e la soluzione commentata: provala nell'editor sul database di esempio (tabelle studenti e classi). Sono gli stessi esercizi di MySQL che assegno ai miei studenti durante le ripetizioni a Roma.

Esercizio 1 — I migliori 3 Base

Mostra i 3 studenti con il voto più alto.

Mostra soluzione
SELECT nome, voto FROM studenti
ORDER BY voto DESC
LIMIT 3;

Esercizio 2 — Studenti di Roma Base

Estrai tutti gli studenti della città di Roma.

Mostra soluzione
SELECT * FROM studenti
WHERE citta = 'Roma';

Esercizio 3 — Conta per città Medio

Conta quanti studenti ci sono per ogni città.

Mostra soluzione
SELECT citta, COUNT(*) AS numero
FROM studenti
GROUP BY citta;

Esercizio 4 — Studenti con aula Medio

Mostra nome studente e aula della sua classe con un INNER JOIN.

Mostra soluzione
SELECT s.nome, c.aula
FROM studenti s
INNER JOIN classi c ON s.classe_id = c.id;

Esercizio 5 — Giudizio con CASE Avanzato

Crea una colonna "giudizio" che vale Ottimo/Sufficiente/Insufficiente in base al voto.

Mostra soluzione
SELECT nome, voto,
  CASE
    WHEN voto >= 8 THEN 'Ottimo'
    WHEN voto >= 6 THEN 'Sufficiente'
    ELSE 'Insufficiente'
  END AS giudizio
FROM studenti;

Preparazione al colloquio — domande MySQL frequenti

Le domande di MySQL più frequenti ai colloqui per ruoli da sviluppatore backend e data analyst.

Differenza tra MyISAM e InnoDB?

InnoDB supporta transazioni e chiavi esterne; MyISAM è più semplice e veloce in sola lettura ma senza integrità referenziale.

Differenza tra WHERE e HAVING?

WHERE filtra le righe prima del raggruppamento; HAVING filtra i gruppi dopo GROUP BY.

Cosa fa AUTO_INCREMENT?

Genera automaticamente un numero progressivo univoco, tipicamente per la chiave primaria.

Come si previene la SQL Injection?

Con i prepared statements / query parametrizzate, mai concatenando l'input dell'utente.
