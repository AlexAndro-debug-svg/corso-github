Introduzione a MySQL

MySQL è uno dei sistemi di gestione di database (DBMS) relazionali più diffusi al mondo, gratuito e open source. È il cuore di moltissimi siti web e applicazioni: è alla base dello stack LAMP (Linux, Apache, MySQL, PHP) e viene gestito spesso tramite l'interfaccia grafica phpMyAdmin.

MySQL usa il linguaggio SQL: tutto ciò che impari qui vale anche per gli altri database relazionali. In questo corso vedremo sia i comandi SQL sia gli aspetti più specifici di MySQL (tipi di dato, AUTO_INCREMENT, funzioni, utenti).

💡 Puoi eseguire le query nel browser: c'è un database di esempio con le tabelle studenti (id, nome, voto, classe, classe_id, citta) e classi (id, aula, classe). Il motore è SQLite, quasi identico a MySQL: dove la sintassi cambia, te lo segnalo.

Cos'è un DBMS

Un DBMS (Database Management System) è il software che memorizza, organizza e protegge i dati, permettendo a più utenti e applicazioni di accedervi contemporaneamente in modo sicuro. MySQL è un DBMS relazionale: i dati sono organizzati in tabelle collegate tra loro tramite chiavi.

  • Database: un contenitore di tabelle (es. il database di una scuola).
  • Tabella: una griglia di righe e colonne (es. studenti).
  • Record (riga): una singola voce (es. lo studente Marco).
  • Campo (colonna): una proprietà (es. il voto).

I tipi di dato in MySQL

Ogni colonna ha un tipo di dato che definisce cosa può contenere. I principali in MySQL:

CategoriaTipiUso
NumericiINT, DECIMAL, FLOATNumeri interi e decimali
TestoVARCHAR(n), TEXT, CHAR(n)Stringhe
Data/oraDATE, DATETIME, TIMESTAMPDate e orari
BooleaniBOOLEAN (TINYINT)Vero/falso
💡 Usa VARCHAR per testi di lunghezza variabile (nomi, email) e INT per gli identificatori. Per importi di denaro preferisci DECIMAL a FLOAT per evitare arrotondamenti.

Creare una tabella: CREATE TABLE

CREATE TABLE definisce una tabella con le sue colonne, i tipi di dato e i vincoli (PRIMARY KEY, NOT NULL, UNIQUE, DEFAULT).

Esempio

AUTO_INCREMENT

Con AUTO_INCREMENT MySQL genera automaticamente un numero progressivo per la chiave primaria a ogni inserimento: non devi indicare tu l'id.

⚙️ In MySQL: id INT AUTO_INCREMENT PRIMARY KEY. In SQLite (l'editor del corso) si usa INTEGER PRIMARY KEY AUTOINCREMENT: il risultato è lo stesso.

Inserire dati: INSERT INTO

INSERT INTO aggiunge una o più righe. Puoi inserire più record in un solo comando elencando più gruppi di VALUES.

Esempio

Leggere i dati: SELECT

SELECT legge i dati da una tabella. Con * ottieni tutte le colonne, oppure elenchi quelle desiderate. DISTINCT elimina i duplicati.

Esempio

Filtrare: WHERE

La clausola WHERE filtra le righe con una condizione. Puoi combinare più condizioni con AND, OR e NOT.

Esempio

Ordinare e limitare: ORDER BY e LIMIT

ORDER BY ordina i risultati (ASC/DESC); LIMIT restituisce solo le prime N righe. Insieme servono per le classifiche e la paginazione.

Esempio

Modificare: UPDATE

UPDATE modifica righe esistenti. Ricorda sempre il WHERE, altrimenti aggiorni tutte le righe!

Esempio

Cancellare: DELETE

DELETE elimina righe. Con il WHERE selezioni quali; senza, svuoti la tabella.

Esempio

Le funzioni di aggregazione

Calcolano un valore su più righe: COUNT() conta, SUM() somma, AVG() media, MIN()/MAX() minimo e massimo.

Esempio

LIKE, IN e BETWEEN

LIKE cerca pattern di testo (% = qualsiasi sequenza); IN verifica l'appartenenza a un elenco; BETWEEN seleziona un intervallo.

Esempio

Unire tabelle: JOIN

Il JOIN combina righe di tabelle collegate. Qui uniamo studenti e classi tramite classe_id. INNER JOIN tiene solo le corrispondenze; LEFT JOIN tiene tutte le righe di sinistra.

Esempio

GROUP BY e HAVING

GROUP BY raggruppa le righe per calcolare aggregati per gruppo; HAVING filtra i gruppi (a differenza di WHERE che filtra le righe).

Esempio

Le funzioni stringa

MySQL offre funzioni per manipolare il testo: UPPER(), LOWER(), LENGTH() e CONCAT() per concatenare.

⚙️ In MySQL si concatena con CONCAT(nome, ' - ', citta). Nell'editor del corso (SQLite) si usa l'operatore ||: il risultato è lo stesso.
Esempio

Le funzioni data e ora

MySQL gestisce date e orari con funzioni come NOW() (data e ora attuali), CURDATE() (data odierna) e YEAR(), MONTH(), DAY() per estrarne le parti.

⚙️ In MySQL: SELECT NOW();. Nell'editor (SQLite) l'equivalente è date('now') e datetime('now').
Esempio

Le viste (VIEW)

Una vista è una tabella virtuale basata su una query salvata: la interroghi come una tabella normale. Semplifica le query complesse e le rende riutilizzabili.

Esempio

Gli indici (INDEX)

Un indice accelera le ricerche su una colonna molto usata nei filtri. Migliora le SELECT al prezzo di inserimenti leggermente più lenti.

Esempio

Utenti e privilegi

MySQL gestisce la sicurezza tramite utenti con permessi specifici. Un amministratore crea utenti e assegna loro solo i privilegi necessari (principio del minimo privilegio). Questi comandi si eseguono sul server MySQL reale.
