Introduzione ad AWS

Amazon Web Services (AWS) è la piattaforma di cloud computing di Amazon, lanciata nel 2006. Offre oltre 200 servizi — server, storage, database, networking, machine learning — disponibili on-demand e pagati a consumo.

DatoValore
Lancio2006 (S3 ed EC2)
Quota di mercato cloud~30-32% (leader mondiale)
ServiziOltre 200
ModelloPay-as-you-go (paghi solo ciò che usi)
🎯 Obiettivo del corso: prepararti alla certificazione AWS Certified Cloud Practitioner (CLF-C02), l'esame d'ingresso che valida la comprensione di base del cloud AWS.

Cloud Computing

Il cloud computing è la fornitura di risorse informatiche (server, storage, database, software) tramite Internet, con pagamento a consumo, invece di possedere data center fisici.

I tre modelli di servizio

ModelloCosa gestisci tuEsempio AWS
IaaS (Infrastructure)SO, applicazioni, datiEC2
PaaS (Platform)Solo applicazioni e datiElastic Beanstalk
SaaS (Software)Niente, usi e bastaAmazon WorkMail

Modelli di deployment

  • Cloud pubblico: risorse condivise erogate da AWS via Internet
  • Cloud privato: infrastruttura dedicata a una sola organizzazione
  • Cloud ibrido: combinazione di cloud e data center on-premise
💡 Pensa al cloud come all'elettricità: non costruisci una centrale, ti colleghi alla rete e paghi i kWh che consumi.

Vantaggi del cloud

VantaggioDescrizione
Nessun investimento inizialeDa CapEx (acquisto hardware) a OpEx (spesa operativa)
ElasticitàScala su e giù in base alla domanda reale
Economia di scalaPrezzi più bassi grazie ai volumi di AWS
Velocità e agilitàNuove risorse in minuti, non settimane
Portata globaleDistribuisci l'app nel mondo in pochi clic
AffidabilitàRidondanza e alta disponibilità integrate
☁️ I 6 vantaggi del cloud secondo AWS: 1) trasformare le spese, 2) beneficiare di economie di scala, 3) smettere di indovinare la capacità, 4) aumentare velocità e agilità, 5) non spendere per gestire data center, 6) diventare globali in minuti.

AWS gestisce data center in tutto il mondo, organizzati in modo gerarchico per garantire prestazioni e resilienza.

ComponenteDescrizione

Availability Zone (AZ)Uno o più data center isolati all'interno di una Region

⚠️ Best practice: distribuisci sempre le risorse su più Availability Zone. Se un'AZ ha un guasto, l'applicazione resta online nelle altre.

Amazon EC2

EC2 (Elastic Compute Cloud) fornisce server virtuali (chiamati istanze) nel cloud. È il servizio IaaS fondamentale: scegli sistema operativo, CPU, RAM e paghi a ore/secondi.

Tipi di istanza

FamigliaOttimizzata per
General Purpose (t, m)Bilanciate, uso generico
Compute Optimized (c)Calcolo intensivo (CPU)
Memory Optimized (r, x)Grandi dataset in memoria
Storage Optimized (i, d)Alto I/O su disco
Accelerated (p, g)GPU, machine learning

Modelli di pricing EC2

ModelloQuando usarlo
On-DemandCarichi variabili e a breve termine
Reserved InstancesCarichi stabili (sconto fino al 72%, impegno 1-3 anni)
Savings PlansImpegno di spesa flessibile
Spot InstancesCapacità inutilizzata (sconto fino al 90%, interrompibile)
AWS CLI — avviare un'istanza EC2
# Avvia un'istanza EC2 t3.micro nella Region di Milano
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t3.micro \
  --key-name la-mia-chiave \
  --region eu-south-1 \
  --count 1

# Elenca le istanze in esecuzione
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running"

Scaling e Load Balancing

La scalabilità è il superpotere del cloud: aggiungere o rimuovere capacità automaticamente in base alla domanda.

ConcettoDescrizione
Scaling verticaleIstanza più potente (più CPU/RAM)
Scaling orizzontalePiù istanze in parallelo (preferito nel cloud)
Auto ScalingAggiunge/rimuove istanze automaticamente
Elastic Load Balancing (ELB)Distribuisce il traffico tra le istanze
⚖️ ELB + Auto Scaling lavorano insieme: il load balancer distribuisce le richieste, l'auto scaling aggiunge istanze quando il traffico cresce e le rimuove quando cala, ottimizzando costi e prestazioni.

Tipi di Load Balancer

  • Application Load Balancer (ALB): livello 7 (HTTP/HTTPS), routing intelligente
  • Network Load Balancer (NLB): livello 4 (TCP/UDP), altissime prestazioni
  • Gateway Load Balancer: per appliance di rete virtuali

Serverless e Container

Con il serverless non gestisci server: scrivi il codice e AWS si occupa di tutto il resto, scalando automaticamente e facendoti pagare solo per l'esecuzione reale.

AWS Lambda

Lambda esegue funzioni in risposta a eventi (una richiesta HTTP, un file caricato su S3, un messaggio). Paghi solo i millisecondi di esecuzione.

AWS Lambda — funzione in Node.js
// Funzione Lambda attivata da un evento (es. API Gateway)
exports.handler = async (event) => {
  const nome = event.queryStringParameters?.nome || 'Mondo';

  return {
    statusCode: 200,
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      messaggio: `Ciao, ${nome}! Eseguito su AWS Lambda.`,
      timestamp: new Date().toISOString()
    })
  };
};

Messaging e integrazione

I servizi di messaging permettono ai componenti di un'applicazione di comunicare in modo disaccoppiato e asincrono, aumentando resilienza e scalabilità.

ServizioModelloUso tipico
SQSCoda di messaggiDisaccoppiare produttori e consumatori
SNSPub/Sub (notifiche)Inviare un messaggio a molti destinatari
EventBridgeEvent busArchitetture event-driven tra servizi
📨 SQS vs SNS: SQS è una coda (un messaggio → un consumatore che lo preleva). SNS è pub/sub (un messaggio → tutti gli iscritti lo ricevono). Spesso si combinano nel pattern "fan-out".
AWS CLI — inviare un messaggio a una coda SQS
aws sqs send-message \
  --queue-url https://sqs.eu-south-1.amazonaws.com/123456789/mia-coda \
  --message-body "Nuovo ordine #4521 da elaborare"

VPC e Networking

Una VPC (Virtual Private Cloud) è la tua rete privata isolata dentro AWS, dove lanci le risorse con pieno controllo su indirizzi IP, subnet e routing.

ComponenteFunzione
VPCRete virtuale isolata
SubnetSottorete (pubblica o privata) in una AZ
Internet GatewayConnette la VPC a Internet
NAT GatewayPermette alle subnet private di uscire su Internet
Security GroupFirewall a livello di istanza (stateful)
Network ACLFirewall a livello di subnet (stateless)
CloudFrontCDN: distribuisce contenuti dalle Edge Location
Route 53Servizio DNS gestito
💡 Subnet pubblica = ha una rotta verso l'Internet Gateway (es. web server). Subnet privata = nessun accesso diretto da Internet (es. database).

Amazon S3 in dettaglio

S3 (Simple Storage Service) archivia oggetti in "bucket" con durabilità del 99,999999999% (11 nove). Le classi di storage ottimizzano i costi in base alla frequenza d'accesso.

Classe S3Uso
StandardDati ad accesso frequente
Intelligent-TieringSposta automaticamente i dati nella classe più economica
Standard-IA / One Zone-IAAccesso infrequente
Glacier / Deep ArchiveArchiviazione a lungo termine, economica
AWS CLI — operazioni su S3
# Crea un bucket
aws s3 mb s3://il-mio-bucket-unico --region eu-south-1

# Carica un file
aws s3 cp ./foto.jpg s3://il-mio-bucket-unico/

# Sincronizza una cartella intera
aws s3 sync ./sito-web s3://il-mio-bucket-unico/

# Elenca il contenuto
aws s3 ls s3://il-mio-bucket-unico/

AWS offre database gestiti per ogni esigenza: relazionali, NoSQL, data warehouse e in-memory.

ServizioTipoUso

AuroraRelazionaleDB cloud-native compatibile MySQL/PostgreSQL

🗄️ RDS vs DynamoDB: scegli RDS per dati strutturati con relazioni e query SQL complesse; scegli DynamoDB per altissima scalabilità, schema flessibile e prestazioni costanti a qualsiasi volume.

Sicurezza e IAM

La sicurezza è la priorità numero uno di AWS. Si basa sul modello di responsabilità condivisa.

ResponsabilitàDi chi
Sicurezza DEL cloud (hardware, rete, data center)AWS
Sicurezza NEL cloud (dati, configurazioni, accessi)Cliente

IAM (Identity and Access Management)

IAM gestisce chi può accedere a cosa. Si basa su utenti, gruppi, ruoli e policy.

IAM — policy JSON che concede sola lettura su un bucket S3
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::il-mio-bucket",
        "arn:aws:s3:::il-mio-bucket/*"
      ]
    }
  ]
}
⚠️ Principio del privilegio minimo: concedi sempre solo i permessi strettamente necessari. Non usare mai l'utente root per le attività quotidiane e attiva l'MFA (autenticazione a più fattori).

📊 CloudWatch vs CloudTrail: CloudWatch monitora le prestazioni (CPU, memoria, traffico). CloudTrail registra le azioni (chi ha avviato un'istanza, chi ha modificato una policy). Entrambi sono essenziali.

Well-Architected Framework

Le 6 colonne per costruire bene sul cloud:

  • 🏆 Operational Excellence — eccellenza operativa
  • 🔒 Security — sicurezza
  • 🛡️ Reliability — affidabilità
  • ⚡ Performance Efficiency — efficienza prestazionale
  • 💰 Cost Optimization — ottimizzazione dei costi
  • 🌱 Sustainability — sostenibilità
🏗️ Il Cloud Adoption Framework (CAF) aiuta le organizzazioni a pianificare la migrazione al cloud organizzando il lavoro in 6 prospettive: Business, People, Governance, Platform, Security, Operations.

Esercizi pratici

🏆 Esercizio 1 — Crea un account Free Tier

Registra un account AWS, attiva l'MFA sull'utente root e crea un utente IAM con permessi limitati. Esplora la console e individua le Region disponibili. (Resta entro il Free Tier per non avere costi.)

🏆 Esercizio 2 — Sito statico su S3

Crea un bucket S3, abilita lo "Static website hosting", carica una semplice pagina HTML e rendila pubblica con una bucket policy. Bonus: metti CloudFront davanti al bucket come CDN.

🏆 Esercizio 3 — Progetta un'architettura

Disegna su carta un'architettura a 3 livelli ad alta disponibilità: ALB + Auto Scaling Group di EC2 su 2 AZ + database RDS Multi-AZ. Indica subnet pubbliche/private e i Security Group.

🏆 Esercizio 4 — Scegli il servizio giusto

Per ognuno di questi scenari indica il servizio AWS più adatto: (a) backup a lungo termine economico, (b) API serverless, (c) cache in-memory, (d) coda per disaccoppiare microservizi, (e) CDN globale.
