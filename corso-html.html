Introduzione a HTML

HTML (HyperText Markup Language) è il linguaggio con cui sono costruite tutte le pagine web. Non è un linguaggio di programmazione: è un linguaggio di marcatura, cioè serve a “etichettare” le parti di un documento dicendo al browser cosa è un titolo, cosa un paragrafo, cosa un'immagine.

💡 In breve: l'HTML descrive la struttura, il CSS l'aspetto (colori, posizioni) e JavaScript il comportamento (interattività).
Esempio

Editor HTML

Per scrivere HTML basta un editor di testo. I più usati sono Visual Studio Code (gratuito), Notepad++ o Sublime Text. Puoi anche usare l'editor “Provalo tu” qui sul sito senza installare nulla.

Un file HTML si salva con estensione .html e si apre con un qualsiasi browser (Chrome, Firefox, Edge).

⭐ Inizia subito: clicca “Provalo tu” su un esempio, modifica il codice e guarda il risultato a destra.

HTML di base

Ogni pagina segue sempre lo stesso schema: dichiarazione del tipo di documento, elemento <html>, una parte <head> (informazioni nascoste) e una parte <body> (il contenuto visibile).

Esempio

Elementi e tag

Un elemento è composto da un tag di apertura, un contenuto e un tag di chiusura: <p> contenuto </p>. Alcuni elementi sono “vuoti” e si chiudono da soli, come <br> e <hr>.

Esempio

Gli attributi

Gli attributi aggiungono informazioni a un elemento e si scrivono nel tag di apertura, nella forma nome="valore".

Esempio

I titoli

I titoli vanno da <h1> (il più importante) a <h6>. Usa <h1> una sola volta per pagina.

Esempio
⭐ I titoli aiutano anche Google a capire di cosa parla la pagina: usali in ordine logico.

I paragrafi

Il testo normale va dentro <p>. Il browser ignora spazi e a capo extra: per andare a capo serve <br>.

Esempio

Gli stili (attributo style)

L'attributo style permette di dare un aspetto a un elemento: colore, sfondo, dimensione del testo, allineamento.

Esempio

Formattare il testo

Tag utili per dare significato al testo:

  • <strong> importante (grassetto), <b> solo grassetto
  • <em> enfasi (corsivo), <i> solo corsivo
  • <mark> evidenziato, <small> piccolo, <del> cancellato, <ins> inserito
  • <sub> pedice, <sup> apice
Esempio

Citazioni

<blockquote> per citazioni lunghe, <q> per citazioni brevi in linea, <abbr> per le abbreviazioni.

Esempio

I commenti

I commenti sono note che il browser ignora. Si scrivono tra <!-- e -->.

Esempio

I colori

I colori si indicano per nome (red), in esadecimale (#ff0000), in RGB (rgb(255,0,0)) o RGBA (con trasparenza).

Esempio

HTML e CSS

Il CSS definisce l'aspetto. Si può scrivere in tre modi: inline (attributo style), interno (dentro <style> nell'head) o esterno (file .css collegato).

Esempio — CSS interno

I link

I link si fanno con <a> e l'attributo href. Con target="_blank" il link si apre in una nuova scheda.

Esempio

Le immagini

Si inseriscono con <img>: l'attributo src indica l'indirizzo e alt il testo alternativo.

Esempio
♿ Accessibilità: l'attributo alt viene letto dai software per non vedenti e mostrato se l'immagine non carica. Mettilo sempre!

La favicon

La favicon è la piccola icona che appare nella scheda del browser. Si aggiunge nell'<head>.

Esempio

Il titolo della pagina

L'elemento <title> nell'head definisce il titolo mostrato nella scheda del browser e nei risultati di Google. È importante per la SEO.

Esempio

Le tabelle

Si costruiscono con <table>. Ogni riga è <tr>, le intestazioni <th> e le celle <td>.

Esempio

Le liste

Liste non ordinate con <ul> (puntini), ordinate con <ol> (numeri). Ogni voce è <li>. Esistono anche le liste di definizione con <dl>.

Esempio

Elementi block e inline

Gli elementi block (come <div>, <p>, <h1>) occupano tutta la larghezza e vanno a capo. Gli elementi inline (come <span>, <a>, <strong>) occupano solo lo spazio necessario e restano nella riga.

Esempio

Div e contenitori

Il <div> è un contenitore generico per raggruppare elementi, soprattutto per dargli uno stile. Lo <span> fa lo stesso ma in linea.

Esempio

Le classi

L'attributo class assegna un nome a uno o più elementi, per applicargli lo stesso stile con il CSS. Più elementi possono avere la stessa classe.

Esempio

L'attributo id

L'id dà un nome unico a un elemento (uno solo per pagina). Si usa per il CSS, per i link interni e per JavaScript. Nel CSS si seleziona con il cancelletto #.

Esempio

Gli iframe

Un <iframe> incorpora un'altra pagina dentro la tua (es. una mappa o un video).

Esempio

HTML e JavaScript

JavaScript rende la pagina interattiva. Il codice va dentro l'elemento <script>. Qui un pulsante che cambia un testo quando lo clicchi.

Esempio

I percorsi dei file

Il percorso indica dove si trova un file. Può essere relativo (rispetto alla pagina) o assoluto (indirizzo completo).

  • img/foto.jpg → cartella img nella stessa posizione
  • ../foto.jpg → cartella superiore
  • https://sito.it/foto.jpg → percorso assoluto

L'elemento head

L'<head> contiene informazioni sulla pagina non visibili: titolo, charset, descrizione (meta), collegamenti a CSS e icone.

Esempio

Il layout della pagina

Una pagina si organizza con elementi semantici: <header> (testata), <nav> (menu), <main> (contenuto), <aside> (barra laterale), <footer> (piè di pagina).

Esempio

HTML responsive

Una pagina responsive si adatta a schermi di ogni dimensione. La regola base è inserire il meta viewport nell'head.

Esempio
⭐ Usa misure flessibili (percentuali, max-width) invece di larghezze fisse in pixel.

Codice nel testo

Per mostrare codice nel testo: <code> (codice in linea), <pre> (testo preformattato, mantiene spazi e a capo), <kbd> (tasti), <samp> (output).

Esempio

HTML semantico

Gli elementi semantici descrivono il significato del contenuto, non solo l'aspetto. Aiutano accessibilità e SEO: <article>, <section>, <figure>, <figcaption>, <time>.

Esempio

Le entità HTML

Alcuni caratteri sono riservati (come < e >) e si scrivono con le entità: &lt; = <, &gt; = >, &amp; = &, &nbsp; = spazio unito, &copy; = ©.

Esempio

I simboli

Con le entità puoi mostrare simboli matematici, valute, frecce: &euro; = €, &deg; = °, &rarr; = →, &hearts; = ♥.

Esempio

Le emoji

Le emoji sono caratteri Unicode: basta che la pagina usi charset="UTF-8" e puoi scriverle direttamente nel testo.

Esempio

Il set di caratteri (charset)

Il charset dice al browser come interpretare i caratteri. Lo standard è UTF-8, che supporta tutte le lingue, le lettere accentate e le emoji. Va sempre messo all'inizio dell'head.

Esempio

La codifica degli URL

Gli URL non possono contenere spazi o caratteri speciali: vengono codificati. Lo spazio diventa %20 (o +), la lettera à diventa %C3%A0.

💡 Per questo nei link delle email scriviamo ?subject=Richiesta%20info: %20 è lo spazio codificato.

I moduli (form)

I form raccolgono dati dall'utente. L'elemento <form> contiene i campi; action indica dove inviare i dati e method come (GET o POST).

Esempio

Attributi del form

Principali attributi: action (URL di destinazione), method (get/post), target (dove aprire la risposta), autocomplete (on/off).

Esempio

Gli elementi del form

Oltre a <input> ci sono: <textarea> (testo lungo), <select> con <option> (menu a tendina), <label> (etichetta), <fieldset> (gruppo).

Esempio

I tipi di input

L'attributo type cambia il tipo di campo: text, email, password, number, date, checkbox, radio, color, range.

Esempio

Attributi degli input

Utili per guidare e validare l'inserimento: placeholder (testo guida), required (obbligatorio), value (valore iniziale), readonly, disabled, maxlength.

Esempio

Canvas (disegnare con il codice)

L'elemento <canvas> è un'area dove disegnare con JavaScript: linee, forme, colori.

Esempio

SVG (grafica vettoriale)

L'<svg> disegna forme vettoriali direttamente in HTML, senza JavaScript. Restano nitide a ogni dimensione.

Esempio

I media in HTML

HTML può mostrare audio e video senza programmi esterni, grazie agli elementi <video> e <audio>. I formati più usati sono MP4 (video) e MP3 (audio).

I video

L'elemento <video> con l'attributo controls mostra i comandi di riproduzione.

Esempio

L'audio

L'elemento <audio> funziona come il video: con controls appaiono i comandi.

Esempio

Incorporare YouTube

Per inserire un video di YouTube si usa un <iframe> con l'indirizzo “embed” del video.

Esempio

Geolocalizzazione

L'API di geolocalizzazione (JavaScript) permette di chiedere la posizione dell'utente — sempre con il suo permesso.

Esempio

Web Storage (memoria del browser)

Con localStorage il browser può salvare dati (come testo) anche dopo aver chiuso la pagina. Utile per ricordare preferenze.

Esempio

Drag and Drop (trascinare)

L'API Drag and Drop permette di trascinare elementi nella pagina. Si attiva con l'attributo draggable="true" e alcuni eventi JavaScript.

Esempio

HTML Web Workers

Un Web Worker è uno script JavaScript che viene eseguito in background, su un thread separato rispetto a quello della pagina. Significa che puoi fare calcoli pesanti senza “congelare” l'interfaccia: l'utente continua a cliccare, scorrere e digitare mentre il worker lavora.

💡 A cosa serve: elaborare grandi quantità di dati, calcoli matematici complessi, parsing di file di grandi dimensioni — tutto senza rendere la pagina lenta o bloccata.

Il worker comunica con la pagina tramite messaggi: si invia con postMessage() e si riceve con l'evento onmessage.

Esempio
⭐ Da ricordare: un Web Worker non può accedere al DOM (non può toccare la pagina direttamente). Restituisce i risultati alla pagina, che poi aggiorna l'HTML.

HTML SSE — Server-Sent Events

Gli Server-Sent Events (SSE) permettono a una pagina web di ricevere aggiornamenti automatici da un server in tempo reale, su un solo canale unidirezionale (dal server al browser). Sono perfetti per notifiche live, quotazioni di borsa, punteggi sportivi, feed di news.

SSEWebSocket
Una direzione (server → browser)Due direzioni (bidirezionale)
Solo testo, su HTTP normaleTesto e binario, protocollo dedicato
Riconnessione automaticaRiconnessione manuale
 
<script>
  const sorgente = new EventSource("/aggiornamenti");
  sorgente.onmessage = function(evento) {
    document.getElementById("notizie").innerHTML += evento.data + "<br>";
  };
</script>
💡 L'oggetto EventSource apre la connessione una sola volta e resta in ascolto: ogni volta che il server invia un dato, scatta onmessage.

Quiz HTML — metti alla prova le tue conoscenze

Un breve quiz di HTML per autovalutarti. Leggi ogni domanda, pensa alla risposta e poi apri la soluzione per verificare.

1. Quale tag definisce il titolo più importante di una pagina?

✅ Il tag <h1>. I titoli vanno da <h1> (il più importante) a <h6>.

2. Come si crea un collegamento (link) a un altro sito?

✅ Con il tag <a href="https://...">testo</a>. L'attributo href indica la destinazione.

3. Quale attributo rende accessibile un'immagine se non si carica?

✅ L'attributo alt, che fornisce un testo alternativo descrittivo.

4. Qual è la differenza tra <div> e <span>?

✅ <div> è un contenitore a blocco (va a capo), <span> è in linea (resta nel flusso del testo).

5. Quale tag HTML5 rappresenta la barra di navigazione?

✅ Il tag semantico <nav>.

Esercizi HTML con soluzioni (per principianti)

Questi esercizi di HTML svolti sono pensati per chi parte da zero e vuole fare pratica con esempi reali. Per ognuno c'è la consegna e la soluzione commentata: prova prima da solo, poi confronta. Sono gli stessi esercizi di HTML che assegno ai miei studenti durante le ripetizioni a Roma.

Esercizio 1 — La tua prima pagina Base

Crea una pagina HTML completa con un titolo principale “Il mio diario” e un paragrafo di presentazione.

Mostra soluzione
<!DOCTYPE html>
<html lang="it">
<head><meta charset="UTF-8"><title>Diario</title></head>
<body>
  <h1>Il mio diario</h1>
  <p>Ciao, qui scrivo i miei pensieri.</p>
</body>
</html>

Crea una lista non ordinata con almeno 4 prodotti della spesa.

Mostra soluzione
<ul>
  <li>Pane</li>
  <li>Latte</li>
  <li>Uova</li>
  <li>Mele</li>
</ul>

Esercizio 3 — Link e immagine Medio

Inserisci un'immagine cliccabile che, se premuta, porti al sito di Wikipedia. Aggiungi sempre il testo alternativo.

Mostra soluzione
<a href="https://it.wikipedia.org">
  <img src="logo.png" alt="Vai a Wikipedia" width="120">
</a>

Esercizio 4 — Tabella orario Medio

Crea una tabella con intestazione (Giorno, Materia) e due righe di dati.

Mostra soluzione
<table border="1">
  <tr><th>Giorno</th><th>Materia</th></tr>
  <tr><td>Lunedì</td><td>Informatica</td></tr>
  <tr><td>Martedì</td><td>Matematica</td></tr>
</table>

Esercizio 5 — Form di iscrizione Avanzato

Crea un form con i campi nome, email (obbligatoria) e un pulsante di invio. Usa le <label> per l'accessibilità.

Mostra soluzione
<form>
  <label>Nome: <input type="text" name="nome"></label><br>
  <label>Email: <input type="email" name="email" required></label><br>
  <button type="submit">Iscriviti</button>
</form>
⭐ Apri ogni soluzione nell'editor “Provalo tu” e modificala: cambiare il codice è il modo migliore per fissare i concetti.

Crea un sito web in HTML (progetto guidato)

Il progetto finale di ogni corso di HTML è costruire un sito web completo. Ecco la struttura minima di un sito multi-pagina che puoi realizzare con quello che hai imparato finora.

Preparazione al colloquio — domande HTML frequenti

Le domande di HTML più frequenti ai colloqui per ruoli web (front-end, web developer junior). Utili anche per le interrogazioni a scuola.

Cos'è l'HTML semantico e perché è importante?

È l'uso di tag che descrivono il significato del contenuto (header, nav, article, footer). Migliora accessibilità e SEO.

Differenza tra elementi block e inline?

Gli elementi block (es. div, p) occupano tutta la larghezza e vanno a capo; gli inline (es. span, a) restano nel flusso del testo.

A cosa serve il DOCTYPE?

Dichiara al browser la versione di HTML usata. <!DOCTYPE html> attiva la modalità standard di HTML5.

Cosa rende un'immagine accessibile?

L'attributo alt con una descrizione testuale significativa.

Riepilogo HTML — i concetti chiave

Un riepilogo veloce di HTML da ripassare prima di una verifica o un colloquio:

  • 📌 L'HTML descrive la struttura di una pagina con i tag.
  • 📌 Ogni pagina ha <!DOCTYPE html>, <html>, <head> e <body>.
  • 📌 Gli attributi aggiungono informazioni ai tag (es. href, src, alt).
  • 📌 I tag semantici (header, nav, main, article, footer) danno significato al contenuto.
  • 📌 I form raccolgono i dati dell'utente.
  • 📌 La pratica è tutto: scrivi codice ogni giorno.

Accessibilità HTML

L'accessibilità web (a11y) significa rendere le pagine usabili da tutti, incluse le persone con disabilità che usano screen reader o navigano con la tastiera. Un buon HTML è già metà del lavoro.

  • ♿ Usa sempre l'attributo alt nelle immagini.
  • ♿ Associa ogni campo del form a una <label>.
  • ♿ Usa i titoli in ordine (h1, poi h2…), senza saltarli.
  • ♿ Indica la lingua con <html lang="it">.
  • ♿ Usa tag semantici al posto di tanti div generici.
  • ♿ Aggiungi attributi ARIA solo quando l'HTML semantico non basta.
⭐ Una pagina accessibile è anche più indicizzata su Google: l'accessibilità aiuta la SEO.

Doctype HTML

Il DOCTYPE è la prima riga di ogni pagina e dice al browser quale versione di HTML usare. In HTML5 è semplicissimo:

<!DOCTYPE html>
💡 Senza DOCTYPE corretto il browser può attivare la “quirks mode”, una modalità di compatibilità che fa rendere la pagina in modo imprevedibile. Mettilo sempre come prima riga.

Set di caratteri (charset)

Il set di caratteri definisce come vengono codificate le lettere. Lo standard moderno è UTF-8, che supporta tutte le lingue ed emoji. Si dichiara nel <head>:

<meta charset="UTF-8">
⭐ Se vedi caratteri strani al posto delle lettere accentate (à, è, ì), quasi sempre manca <meta charset="UTF-8">.
