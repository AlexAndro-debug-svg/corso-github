Introduzione a Bash

Bash (Bourne Again SHell) è l'interprete di comandi più diffuso nei sistemi Unix/Linux e macOS. Permette di eseguire comandi, automatizzare operazioni ripetitive e scrivere script potenti per la gestione del sistema.

Bash è uno strumento fondamentale per:

  • Amministratori di sistema che automatizzano task quotidiani
  • Sviluppatori che gestiscono build, deploy e ambienti CI/CD
  • Data scientist che elaborano file di testo e dati
  • Chiunque voglia usare Linux/macOS in modo efficiente

In questo corso imparerai i comandi fondamentali, il text processing, il monitoraggio del sistema, il networking, la gestione dei file e la scrittura di script Bash completi.

Comando ls

ls elenca il contenuto di una directory.

Sintassi

ls [opzioni] [percorso]

Utilizzo base

ls          # directory corrente
ls /etc     # directory specificata
ls -l       # formato lungo (dettagliato)
ls -a       # mostra file nascosti
ls -la      # lungo + nascosti
ls -lh      # dimensioni leggibili (KB, MB)
ls -lt      # ordina per data di modifica
ls -lS      # ordina per dimensione

Output del formato lungo

-rw-r--r-- 1 user group  4096 Jun 12 10:00 file.txt
drwxr-xr-x 2 user group  4096 Jun 12 09:00 cartella/

Le colonne: permessi, link, proprietario, gruppo, dimensione, data, nome.

Altre opzioni

ls -R       # ricorsivo (include sottocartelle)
ls *.txt    # solo file .txt
ls -1       # un file per riga

Comando pwd

pwd (print working directory) mostra il percorso assoluto della directory corrente.

Comando cp

cp copia file e directory.

Sintassi

cp [opzioni] sorgente destinazione

Copia file

cp file.txt copia.txt              # copia con nuovo nome
cp file.txt /tmp/                  # copia in altra directory
cp file1.txt file2.txt /backup/    # copia più file

Copia directory

cp -r cartella/ backup/     # -r = ricorsivo (obbligatorio per dir)
cp -r src/ /tmp/src_backup/

Opzioni utili

cp -i file.txt dest.txt     # chiede conferma se dest esiste
cp -u file.txt dest.txt     # copia solo se sorgente è più recente
cp -v file.txt dest.txt     # verbose
cp -p file.txt dest.txt     # preserva permessi e timestamp
cp -a cartella/ backup/     # archivio: -r + -p + symlink

Comando mv

mv sposta o rinomina file e directory.

Rinominare

Spostare

mv file.txt /tmp/              # sposta in /tmp
mv *.log /var/log/archivio/   # sposta tutti i .log

Opzioni

mv -i src dest    # chiede conferma prima di sovrascrivere
mv -n src dest    # non sovrascrive mai (no-clobber)
mv -v src dest    # verbose

Rinomina batch con ciclo

Comando grep

grep cerca pattern all'interno di file o input. È uno dei comandi più potenti di Bash.

Sintassi

grep [opzioni] pattern [file...]

Utilizzo base

grep "errore" log.txt              # cerca "errore" in log.txt
grep "root" /etc/passwd            # cerca utente root
grep "TODO" *.py                   # cerca in tutti i .py

Opzioni fondamentali

grep -i "errore" log.txt    # case-insensitive
grep -r "TODO" ./src/       # ricorsivo nelle directory
grep -n "errore" log.txt    # mostra numero di riga
grep -c "errore" log.txt    # conta le righe
grep -v "errore" log.txt    # INVERSO: righe senza match
grep -l "TODO" *.py         # solo nomi dei file
grep -w "log" file.txt      # solo parole intere

Espressioni regolari

grep "^#" script.sh          # righe che iniziano con #
grep "\.py$" lista.txt       # righe che finiscono con .py
grep -E "err(ore|or)" log.txt # pattern esteso
grep -P "\d{3}-\d{4}" file.txt  # Perl regex

Con pipe

ps aux | grep python         # processi Python
history | grep git           # comandi git usati
cat /etc/passwd | grep -v "nologin"  # utenti con shell

Comando awk

awk è un linguaggio di elaborazione di testo orientato ai campi (colonne). Perfetto per file CSV, log e output di comandi.

Sintassi

awk 'pattern {azione}' file

Variabili speciali

# $0 = riga intera, $1 = primo campo, ecc.
# NR = numero di riga, NF = numero di campi

echo "Mario Rossi 42" | awk '{print $1}'    # Mario
echo "Mario Rossi 42" | awk '{print $2}'    # Rossi
echo "Mario Rossi 42" | awk '{print $NF}'   # 42 (ultimo campo)

Separator personalizzato

awk -F: '{print $1}' /etc/passwd        # utenti
awk -F, '{print $1, $3}' dati.csv       # colonne 1 e 3 da CSV
awk -F: '{print $1, $3}' /etc/passwd    # utente e UID

Filtri e condizioni

awk 'NR>1' file.csv               # salta intestazione
awk '$3 > 100' vendite.txt        # righe dove col3 > 100
awk '/errore/ {print NR, $0}' log.txt

Calcoli

awk '{sum += $1} END {print "Totale:", sum}' numeri.txt
awk 'END {print "Righe:", NR}' file.txt
ps aux | awk 'NR>1 {sum+=$3} END {print "CPU%:", sum}'

Comando sed

sed (stream editor) elabora e trasforma testo riga per riga. Ideale per sostituzioni in massa.

Sostituzione

sed 's/vecchio/nuovo/' file.txt        # prima occorrenza per riga
sed 's/vecchio/nuovo/g' file.txt       # tutte le occorrenze
sed 's/vecchio/nuovo/gi' file.txt      # case-insensitive
sed -i 's/vecchio/nuovo/g' file.txt    # modifica il file direttamente

Backup automatico

sed -i.bak 's/foo/bar/g' file.txt   # salva backup in file.txt.bak

Eliminare righe

sed '5d' file.txt            # elimina riga 5
sed '1,3d' file.txt          # elimina righe 1-3
sed '/^#/d' script.sh        # elimina commenti
sed '/^$/d' file.txt         # elimina righe vuote

Stampa selettiva

sed -n '10,20p' file.txt     # stampa solo righe 10-20
sed -n '/errore/p' log.txt  # stampa righe con "errore"

Inserire testo

sed '1i\# Intestazione aggiunta' file.txt   # prima della riga 1
sed '$a\# Fine file' file.txt              # alla fine

Comandi kill e uptime

kill — Terminare processi

Ogni processo ha un PID (Process ID). kill invia segnali ai processi.

kill 1234          # invia SIGTERM al PID 1234 (terminazione gentile)
kill -9 1234       # SIGKILL (terminazione forzata, immediata)
kill -15 1234      # SIGTERM (default)
kill -l            # elenco di tutti i segnali

Trovare il PID

pgrep -u utente    # PID dei processi dell'utente
ps aux | grep app  # cerca per nome

killall e pkill

killall firefox          # termina tutti i processi "firefox"
pkill -u utente         # termina tutti i processi di un utente
pkill -f "python app"   # cerca nel comando completo

Compressione File

zip e unzip

zip archivio.zip file1.txt file2.txt    # comprime
zip -r archivio.zip cartella/          # ricorsivo
zip -9 archivio.zip file.txt           # massima compressione
unzip archivio.zip                     # estrae
unzip archivio.zip -d /dest/           # estrae in directory
unzip -l archivio.zip                  # elenca contenuto

tar — Archivio (con o senza compressione)

# Creare archivi
tar -cvf archivio.tar cartella/          # solo archivio

tar -cjvf archivio.tar.bz2 cartella/    # + bzip2

# Elencare contenuto
tar -tvf archivio.tar.gz

Permessi File

Ogni file ha tre livelli di permessi: proprietario, gruppo e altri. Ogni livello ha tre bit: read (4), write (2), execute (1).

Lettura dei permessi

-rwxr-xr-- 1 mario devs 4096 Jun 12 10:00 script.sh
# rwx = proprietario: legge, scrive, esegue
# r-x = gruppo: legge, esegue
# r-- = altri: solo legge

chmod — Cambia permessi

# Notazione ottale
chmod 755 script.sh      # rwxr-xr-x
chmod 644 file.txt       # rw-r--r--
chmod 600 chiave.pem     # rw------- (solo proprietario)

# Notazione simbolica
chmod +x script.sh       # aggiunge esecuzione a tutti
chmod u+w file.txt       # aggiunge scrittura al proprietario
chmod g-w file.txt       # rimuove scrittura dal gruppo
chmod o-r file.txt       # rimuove lettura dagli altri
chmod -R 755 cartella/   # ricorsivo

chown — Cambia proprietario

chown mario file.txt             # cambia proprietario
chown mario:devs file.txt        # cambia proprietario e gruppo
chown -R mario:devs cartella/    # ricorsivo

chgrp — Cambia gruppo

chgrp devs file.txt
chgrp -R devs cartella/

Script Bash

Uno script Bash è un file di testo con comandi eseguiti in sequenza. Permettono di automatizzare operazioni ripetitive.

Struttura base

#!/bin/bash
# Questo è un commento
# La prima riga (shebang) indica l'interprete

echo "Il mio primo script!"
date
whoami

Rendere eseguibile

chmod +x mio_script.sh    # aggiunge permesso di esecuzione
./mio_script.sh           # esegue
bash mio_script.sh        # alternativa

Argomenti da riga di comando

#!/bin/bash
echo "Nome script: $0"
echo "Primo argomento: $1"
echo "Secondo argomento: $2"
echo "Tutti gli argomenti: $@"
echo "Numero di argomenti: $#"

# Esecuzione: ./script.sh pippo pluto
# Output:
# Nome script: ./script.sh
# Primo argomento: pippo

Codice di uscita

ls file_inesistente.txt
echo "Codice uscita: $?"   # 0 = successo, ≠0 = errore

exit 0     # uscita con successo
exit 1     # errore generico

Variabili in Bash

Definizione e uso

NOME="Mario"
ETA=30

echo $NOME          # Mario
echo "$ETA anni"    # 30 anni
echo "${NOME}!"     # Mario!

Variabili speciali

$HOME      # directory home
$USER      # nome utente corrente
$PATH      # percorsi degli eseguibili
$PWD       # directory corrente
$SHELL     # shell in uso
$RANDOM    # numero casuale 0-32767
$$         # PID dello script

Sostituzione di comando

DATA=$(date +%Y-%m-%d)
echo "Oggi è: $DATA"

NUM_FILE=$(ls | wc -l)
echo "File nella directory: $NUM_FILE"

readonly e unset

readonly VERSIONE="1.0"
# VERSIONE="2.0"   # errore!

TEMP="valore"
unset TEMP
echo $TEMP          # stringa vuota

Tipi di Dati in Bash

Bash tratta tutto come stringhe. Per numeri interi usa declare -i o aritmetica esplicita.

Stringhe

STR="Ciao, mondo"
echo ${#STR}              # lunghezza: 11
echo ${STR:0:4}           # substring: Ciao
echo ${STR/Ciao/Hello}    # sostituzione: Hello, mondo
echo ${STR^^}             # maiuscolo: CIAO, MONDO
echo ${STR,,}             # minuscolo: ciao, mondo

Interi

declare -i NUM=10
NUM=NUM+5
echo $NUM        # 15

((risultato = 10 * 3 + 2))
echo $risultato  # 32

echo $((100 / 7))    # 14 (divisione intera)
echo $((100 % 7))    # 2 (modulo)

Array

FRUTTI=("mela" "banana" "ciliegia")
echo ${FRUTTI[0]}     # mela
echo ${FRUTTI[@]}     # tutti gli elementi
echo ${#FRUTTI[@]}    # numero di elementi: 3

FRUTTI+=("pera")      # aggiunge elemento

for f in "${FRUTTI[@]}"; do
    echo "  - $f"
done

Array associativi

declare -A CAPITALI
CAPITALI["italia"]="Roma"
CAPITALI["francia"]="Parigi"
echo ${CAPITALI["italia"]}   # Roma
echo ${!CAPITALI[@]}         # chiavi

Operatori in Bash

Operatori aritmetici

echo $((5 + 3))   # 8
echo $((5 - 3))   # 2
echo $((5 * 3))   # 15
echo $((5 / 2))   # 2 (divisione intera)
echo $((5 % 2))   # 1 (modulo)
echo $((2 ** 8))  # 256 (potenza)

Confronto numeri (in [ ] e [[ ]])

[ $A -eq $B ]   # uguale (equal)
[ $A -ne $B ]   # diverso (not equal)
[ $A -lt $B ]   # minore (less than)
[ $A -le $B ]   # minore o uguale
[ $A -gt $B ]   # maggiore (greater than)
[ $A -ge $B ]   # maggiore o uguale

Operatori su stringhe

[ "$A" = "$B" ]     # uguale
[ "$A" != "$B" ]    # diverso
[ -z "$A" ]         # stringa vuota
[ -n "$A" ]         # stringa non vuota
[[ "$A" == *"sub"* ]]  # contiene sottostringa

Operatori su file

[ -e percorso ]    # esiste
[ -f percorso ]    # è un file regolare
[ -d percorso ]    # è una directory
[ -r percorso ]    # è leggibile
[ -w percorso ]    # è scrivibile
[ -x percorso ]    # è eseguibile
[ -s percorso ]    # esiste e non è vuoto

If / Else in Bash

Struttura base

if [ condizione ]; then
    # comandi se vera
elif [ altra_condizione ]; then
    # comandi se altra vera

    echo "Contatore: $CONTATORE"
    ((CONTATORE++))

break e continue

Esempio completo — logger

#!/bin/bash

log() {
    local livello=$1
    local messaggio=$2
    echo "[$(date '+%H:%M:%S')] [$livello] $messaggio"
}

verifica_file() {
    local percorso=$1
    if [ ! -f "$percorso" ]; then
        log "ERRORE" "File non trovato: $percorso"
        return 1
    fi
    log "INFO" "File trovato: $percorso"
    return 0
}

verifica_file "/etc/passwd" && log "OK" "Tutto bene"

Cron — Pianificazione dei Task

cron esegue comandi automaticamente a intervalli pianificati tramite crontab.

Gestire il crontab

crontab -e     # apre l'editor del crontab
crontab -l     # elenca i job configurati
crontab -r     # rimuove tutti i job

Formato crontab

# minuto  ora  giorno  mese  giorno_settimana  comando
# (0-59) (0-23) (1-31) (1-12)    (0-7, 0=dom)

0 9 * * 1-5  /home/user/backup.sh    # Lun-Ven alle 09:00
30 2 * * *   /usr/bin/backup         # ogni giorno alle 02:30
*/15 * * * * /scripts/check.sh       # ogni 15 minuti
0 0 1 * *    /scripts/monthly.sh     # primo del mese a mezzanotte
@reboot      /scripts/startup.sh     # all'avvio del sistema

Scorciatoie speciali

@reboot    # all'avvio
@hourly    # ogni ora
@daily     # ogni giorno
@weekly    # ogni settimana
@monthly   # ogni mese
@yearly    # ogni anno

Esercizio 1 — Backup automatico

Scrivi uno script che faccia il backup di una directory, nomini l'archivio con la data e mantenga solo gli ultimi 7 backup.

#!/bin/bash
SORGENTE="$1"
DEST_DIR="$HOME/backup"
DATA=$(date +%Y%m%d_%H%M%S)

mkdir -p "$DEST_DIR"
tar -czvf "$DEST_DIR/backup_$DATA.tar.gz" "$SORGENTE"

# Mantieni solo gli ultimi 7
ls -t "$DEST_DIR"/backup_*.tar.gz | tail -n +8 | xargs rm -f
echo "Backup completato."

#!/bin/bash
LOG="$1"

grep -i "error" "$LOG" | \
    awk '{$1=$2=$3=""; print $0}' | \
    sort | \
    uniq -c | \
    sort -rn | \
    head -5

Esercizio 4 — Rinomina batch

Rinomina tutti i file .jpeg in .jpg in una directory.

#!/bin/bash
DIR="${1:-.}"
CONTATORE=0

for file in "$DIR"/*.jpeg; do
    [ -f "$file" ] || continue
    nuovo="${file%.jpeg}.jpg"
    mv "$file" "$nuovo"
    echo "Rinominato: $(basename $file)"
    ((CONTATORE++))
done
echo "Totale rinominati: $CONTATORE"
