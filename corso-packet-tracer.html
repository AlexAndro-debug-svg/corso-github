Cos'è Cisco Packet Tracer

Cisco Packet Tracer è un software gratuito di simulazione di reti sviluppato da Cisco, pensato per studenti e per chi prepara la certificazione CCNA. Permette di costruire reti virtuali trascinando dispositivi (router, switch, PC), collegarli con i cavi e configurarli esattamente come faresti con apparati reali, ma senza spendere nulla e senza rischi.

È lo strumento ideale per imparare le reti in modo pratico: puoi vedere i pacchetti viaggiare, simulare guasti e capire davvero come comunicano i computer.

📥 Si scarica gratuitamente registrandosi alla Cisco Networking Academy (netacad.com).

Le reti di computer

Una rete è un insieme di dispositivi collegati che si scambiano dati. Le reti si classificano per dimensione:

  • LAN (Local Area Network): rete locale, es. un'aula o un ufficio.
  • MAN (Metropolitan Area Network): copre una città.
  • WAN (Wide Area Network): copre grandi distanze; Internet è la WAN più grande.

I dati viaggiano divisi in pacchetti, piccole unità che attraversano la rete e vengono ricomposte a destinazione.

I dispositivi di rete

DispositivoFunzione
SwitchCollega i dispositivi di una stessa LAN, inoltrando i dati solo al destinatario (usa gli indirizzi MAC).
RouterCollega reti diverse e instrada i pacchetti tra di esse (usa gli indirizzi IP). È la “porta” verso Internet.
Access PointFornisce la connettività Wi-Fi.
HubDispositivo obsoleto: ritrasmette i dati a tutti, meno efficiente dello switch.

Il modello OSI

Il modello OSI (Open Systems Interconnection) descrive la comunicazione di rete in 7 livelli, dal cavo fisico fino all'applicazione. Capirlo aiuta a localizzare i problemi.

LivelloNomeEsempio
7ApplicazioneHTTP, FTP, DNS
6PresentazioneCrittografia, formati
5SessioneGestione connessioni
4TrasportoTCP, UDP
3ReteIP, router
2CollegamentoMAC, switch
1FisicoCavi, segnali
💡 Un modo per ricordarli (dal 7 all'1): “Alcune Persone Sembrano Trovare Risposte Carine e Furbe”.

Indirizzi IP e subnet mask

Un indirizzo IP identifica un dispositivo nella rete. Nella versione IPv4 è formato da 4 numeri (0-255) separati da punti, es. 192.168.1.10. La subnet mask (es. 255.255.255.0) indica quale parte dell'indirizzo identifica la rete e quale l'host.

  • Indirizzo di rete: identifica la rete (es. 192.168.1.0).
  • Gateway: l'indirizzo del router, “uscita” verso le altre reti.
  • IP privati: 192.168.x.x, 10.x.x.x, 172.16-31.x.x (usati nelle LAN).
🔢 Con subnet mask /24 (255.255.255.0) una rete ha 254 indirizzi utilizzabili per gli host.

Creare una LAN in Packet Tracer

Per costruire la tua prima rete locale in Packet Tracer:

  1. Trascina nell'area di lavoro 2 PC e 1 switch.
  2. Collegali con il cavo diritto (Copper Straight-Through) alle porte FastEthernet.
  3. Clicca su ogni PC → scheda Desktop → IP Configuration e assegna gli indirizzi IP.
DispositivoIPSubnet mask
PC0192.168.1.10255.255.255.0
PC1192.168.1.11255.255.255.0

Ora i due PC, essendo nella stessa rete, possono comunicare tramite lo switch.

La CLI di Cisco IOS

Router e switch Cisco si configurano tramite la CLI (Command Line Interface) del sistema operativo IOS, dalla scheda CLI del dispositivo. Esistono diverse modalità, riconoscibili dal prompt:

Cisco IOS — modalità
! Modalità utente (sola lettura)
Router>
! Modalità privilegiata (enable)
Router> enable
Router#
! Modalità di configurazione globale
Router# configure terminal
Router(config)#

Configurare uno switch

Una configurazione di base assegna un nome allo switch e protegge l'accesso con una password.

Cisco IOS
Switch> enable
Switch# configure terminal
Switch(config)# hostname SW-Aula1
SW-Aula1(config)# enable secret cisco123
SW-Aula1(config)# exit
SW-Aula1# write memory   ! salva la configurazione

Configurare un router

Il compito principale del router è dare un indirizzo IP alle sue interfacce, che faranno da gateway per le reti collegate.

Cisco IOS
Router> enable
Router# configure terminal
Router(config)# hostname R1
R1(config)# interface gigabitEthernet 0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown   ! accende l'interfaccia
R1(config-if)# exit
💡 Imposta poi su ogni PC il gateway predefinito 192.168.1.1 per uscire dalla LAN.

Le VLAN

Una VLAN (Virtual LAN) divide logicamente uno switch in più reti separate, anche se i dispositivi sono fisicamente connessi allo stesso switch. Migliora sicurezza e organizzazione (es. VLAN “Studenti” e VLAN “Docenti”).

Cisco IOS
SW(config)# vlan 10
SW(config-vlan)# name Studenti
SW(config-vlan)# exit
SW(config)# interface fastEthernet 0/1
SW(config-if)# switchport mode access
SW(config-if)# switchport access vlan 10

Il routing statico

Per far comunicare due reti diverse, il router deve sapere come raggiungerle. Con il routing statico l'amministratore inserisce manualmente le rotte. (In alternativa esistono i protocolli dinamici come OSPF e RIP.)

Cisco IOS
! Per raggiungere la rete 192.168.2.0 passa dal router 10.0.0.2
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

Il DHCP

Il DHCP (Dynamic Host Configuration Protocol) assegna automaticamente gli indirizzi IP ai dispositivi della rete, così non devi configurarli a mano uno per uno. Si può attivare sul router.

Cisco IOS
R1(config)# ip dhcp pool LAN_AULA
R1(dhcp-config)# network 192.168.1.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.1.1
R1(dhcp-config)# dns-server 8.8.8.8

Sui PC basta impostare IP Configuration → DHCP per ricevere automaticamente l'indirizzo.

Comandi di verifica

Per controllare che la rete funzioni, si usano comandi di diagnostica dai PC e dai dispositivi:

Comandi di test
! Dal prompt dei comandi di un PC:
ping 192.168.1.11      ! verifica la connettività
ipconfig              ! mostra la configurazione IP
! Dalla CLI di un router/switch:
R1# show ip interface brief   ! stato delle interfacce
R1# show running-config       ! configurazione attiva
R1# show ip route             ! tabella di routing
💡 Se il ping riceve risposta, la comunicazione funziona. Usa anche la modalità Simulation di Packet Tracer per vedere i pacchetti viaggiare passo passo.

Complimenti! 🎉 Hai completato il corso di Packet Tracer

Hai imparato i concetti fondamentali delle reti (LAN/WAN, modello OSI, indirizzi IP), i dispositivi di rete e la configurazione pratica di switch e router con la CLI Cisco IOS, incluse VLAN, routing statico e DHCP.
