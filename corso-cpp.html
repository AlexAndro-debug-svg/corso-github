Introduzione a C++

C++ è un linguaggio creato da Bjarne Stroustrup nel 1985 come estensione del linguaggio C, al quale aggiunge la programmazione a oggetti. Unisce l'efficienza e il controllo di basso livello del C con strumenti di astrazione di alto livello.

È il linguaggio di riferimento per videogiochi (Unreal Engine), motori grafici, applicazioni desktop, software scientifico, sistemi real-time e tutto ciò che richiede massime prestazioni. Padroneggiare C++ significa avere il pieno controllo del computer mantenendo un codice organizzato.

📌 C++ non si esegue nel browser: serve un compilatore (es. g++). In questo corso gli esempi si leggono e si modificano con la sintassi colorata.

Le funzioni

Le funzioni in C++ possono avere parametri di default e si possono sovraccaricare (overloading): stesso nome, parametri diversi.

Esempio

Riferimenti e puntatori

Un riferimento (&) è un alias di una variabile; un puntatore (*) ne contiene l'indirizzo. Servono per modificare variabili dentro le funzioni ed evitare copie costose.

Esempio

I puntatori

Un puntatore contiene l'indirizzo di memoria di un'altra variabile. Si dichiara con * e si ottiene l'indirizzo con &. A differenza dei riferimenti, può essere null e riassegnato.

Esempio
📌 Riferimento vs puntatore: il riferimento è un alias fisso e sempre valido; il puntatore è più flessibile (può essere null e riassegnato) ma va usato con attenzione.

Gestione della memoria

In C++ si alloca memoria dinamicamente con new e si libera con delete. Il C++ moderno preferisce gli smart pointer (unique_ptr, shared_ptr) che liberano la memoria automaticamente.

new / delete
💡 Regola d'oro: ogni new deve avere il suo delete. Per evitare i memory leak, nel C++ moderno usa sempre gli smart pointer.

Parametri delle funzioni

I parametri possono essere passati per valore (copia), per riferimento (&, modifica l'originale) o con valori di default.

Scope delle variabili

Programmazione a oggetti (OOP)

La grande novità di C++ rispetto al C è la programmazione orientata agli oggetti: il codice si organizza in classi e oggetti. I pilastri sono incapsulamento, ereditarietà, polimorfismo e astrazione.

📦 Una classe è lo “stampo”, gli oggetti sono i prodotti creati da quello stampo.

Classi e oggetti

Una classe contiene attributi e metodi. Le sezioni public e private ne controllano la visibilità.

Esempio

Metodi di classe

I metodi sono le funzioni di una classe. Possono essere definiti dentro la classe o fuori usando l'operatore di scope ::.

Esempio

I costruttori

Il costruttore ha lo stesso nome della classe e inizializza l'oggetto al momento della creazione.

Esempio

L'incapsulamento

Si nascondono gli attributi rendendoli private e si espongono metodi pubblici (getter/setter) per accedervi in modo controllato.

Friend functions

Una funzione friend (amica) può accedere ai membri private e protected di una classe, pur non essendone un metodo. Si dichiara con la parola chiave friend.

Esempio

Gestione errori in C++

Gli errori in C++ si dividono in tre tipi: sintassi (rilevati dal compilatore), runtime (durante l'esecuzione) e logici (risultati sbagliati). Le eccezioni gestiscono gli errori a runtime in modo controllato.

  • Errori di sintassi: punto e virgola mancante, parentesi non chiuse — bloccano la compilazione
  • Errori a runtime: divisione per zero, accesso fuori dai limiti, puntatori nulli
  • Errori logici: il programma gira ma produce un risultato errato
💡 Compila con g++ -Wall -g programma.cpp per attivare tutti gli avvisi e il debug, poi usa gdb o valgrind per analizzare gli errori a runtime.

Debugging in C++

Il debugging individua e corregge gli errori. Gli strumenti principali sono cout di controllo, il debugger GDB, Valgrind per la memoria e i debugger integrati negli IDE (Visual Studio, CLion, VS Code).

Debug con cerr
📌 cerr è il flusso di errore standard: utile per i messaggi di debug, separati dall'output normale (cout).

Eccezioni (try / catch / throw)

Le eccezioni gestiscono gli errori a runtime senza interrompere bruscamente il programma. Si lancia un'eccezione con throw e si cattura con try/catch.

Quiz C++ — verifica le tue conoscenze

  1. Differenza tra C e C++?
    Risposta: C++ estende C aggiungendo la programmazione a oggetti (classi), i template, le eccezioni, l'overloading e la STL.
  2. Cos'è un riferimento (&)?
    Risposta: Un alias di una variabile esistente. A differenza del puntatore non può essere null né riassegnato, e si usa con la sintassi normale.
  3. Differenza tra struct e class?
    Risposta: Tecnicamente solo la visibilità di default: public nelle struct, private nelle class. Per convenzione struct si usa per dati semplici.
  4. Cos'è il polimorfismo?
    Risposta: La capacità di trattare oggetti di classi derivate tramite un puntatore/riferimento alla classe base, con metodi virtual risolti a runtime.
  5. A cosa serve la STL?
    Risposta: La Standard Template Library fornisce container (vector, map, set), iteratori e algoritmi generici pronti all'uso, basati sui template.

Domande da colloquio C++

  • 💬 Differenza tra riferimento e puntatore? — Un riferimento è un alias (non può essere null né riassegnato); un puntatore è una variabile che contiene un indirizzo e può essere riassegnato.
  • 💬 Cos'è un metodo virtual? — Un metodo che può essere ridefinito nelle classi derivate e risolto a runtime (late binding), base del polimorfismo.
  • 💬 Cosa sono gli smart pointer? — unique_ptr, shared_ptr, weak_ptr: gestiscono automaticamente la memoria evitando i memory leak (RAII).
  • 💬 Cos'è il RAII? — Resource Acquisition Is Initialization: le risorse si acquisiscono nel costruttore e si rilasciano nel distruttore, garantendo la pulizia automatica.
  • 💬 Differenza tra vector e list? — vector è un array dinamico (accesso casuale O(1), inserimento in mezzo O(n)); list è doppiamente concatenata (inserimento O(1), niente accesso casuale).
  • 💬 Cos'è l'overloading degli operatori? — Permette di ridefinire il comportamento degli operatori (+, ==, <<) per i tipi personalizzati.
